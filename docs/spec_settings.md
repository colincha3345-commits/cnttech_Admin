# 시스템 설정(Settings) 기획 명세서

본 문서는 관리자 대시보드 내 **설정** 페이지의 프론트엔드 및 백엔드 개발 시 필요한 세부 필수 항목과 페이지 프로세스를 정의한 문서다.

---

## 1. 라우트 구조

| 경로 | 페이지 | 권한 |
| :--- | :--- | :--- |
| `/settings` | 시스템 설정 (Settings) | settings:read |

---

## 2. 페이지 프로세스

1. **일반 설정** — 서비스명, 로고, 연락처, 이메일 등 브랜드 기본 정보를 관리한다.
2. **알림 설정** — 관리자 알림(이메일/Slack) 수신 여부와 채널을 설정한다.
3. **보안 설정** — 비밀번호 정책, 세션 타임아웃, MFA 강제 여부 등을 관리한다.
4. **정산 설정** — 기본 수수료율, 정산 주기(반월/월), 지급 요일 등을 설정한다.

---

## 3. 세부 개발 명세

### 3.1. 프론트엔드

| 기능 / 필드명 | 입력/노출 형태 | 필수 여부 | 비고 |
| :--- | :--- | :---: | :--- |
| **서비스명** | Input | Y | 최대 50자이다. |
| **로고 이미지** | File Upload | N | 5MB 이하이다. |
| **대표 연락처** | Input | Y | 전화번호 포맷이다. |
| **대표 이메일** | Input | Y | 이메일 형식이다. |
| **세션 타임아웃 (분)** | Number Input | Y | 5~120분이다. |
| **MFA 강제 여부** | Switch | Y | 전체 관리자에게 MFA 강제이다. |
| **기본 수수료율 (%)** | Number Input | Y | 0~100이다. |
| **정산 주기** | Select | Y | biweekly/monthly이다. |

### 3.2. 백엔드

#### API 엔드포인트

| Method | Path | 설명 |
| :--- | :--- | :--- |
| GET | `/api/settings` | 전체 설정 조회이다. |
| PUT | `/api/settings` | 설정 저장. 감사 로그 기록이다. |

**[비즈니스 로직 제약사항]**
- 설정 변경 시 감사 로그를 기록한다(변경 전/후 스냅샷).
- admin 역할만 설정 변경이 가능하다.
- 세션 타임아웃 변경 시 현재 활성 세션에는 즉시 적용하지 않고, 다음 로그인부터 적용한다.

**[⚠️ 트래픽/성능 검토]**
- 설정 데이터는 단일 레코드이므로 Redis에 캐싱하고, 변경 시 캐시를 즉시 갱신한다. 모든 API에서 수수료율/세션 정책을 참조하므로 캐시 히트율이 높아야 한다.

## 4. 정상작동 시나리오

### 시나리오 1: 시스템 설정 변경

| 단계 | 사용자 행동 | 시스템 응답 | 검증 포인트 |
| :---: | :--- | :--- | :--- |
| 1 | 설정 페이지 진입 | 현재 설정값 로드 | 각 섹션별 현재 값 |
| 2 | 설정값 변경 | 실시간 유효성 검증 | 범위/형식 체크 |
| 3 | [저장] 클릭 | `PUT /api/settings` → 성공 Toast | 감사 로그 기록 |
| 4 | 변경 사항 즉시 적용 | 시스템 전역에 반영 | 캐시 무효화 확인 |
