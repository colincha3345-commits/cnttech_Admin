# 시스템 설정(Settings) 기획 명세서

본 문서는 관리자 대시보드 내 **설정** 페이지의 프론트엔드 및 백엔드 개발 시 필요한 세부 필수 항목과 페이지 프로세스를 정의한 문서다.

---

## 1. 라우트 구조

| 경로 | 페이지 | 권한 |
| :--- | :--- | :--- |
| `/settings` | 시스템 설정 (Settings) | settings:read |

---

## 2. 페이지 프로세스

1. **브랜드 기본 정보 조회** — 슈퍼어드민(본사)에서 초기 세팅한 브랜드명, 로고 등 기본 정보를 조회한다. (수정 불가)
2. **운영 유형 및 정책 조회** — 주문 유형 세팅값, 배달비/최소주문금액, 운영 요금제 정책을 확인한다. (수정 불가)
3. **업체 연동 및 계약 정보 조회** — PG사, POS 연동 상태 및 계약 정보, 수수료, 매니저 연락처를 확인한다. (수정 불가)
4. **가이드 및 고객 지원** — 사이트 기능 사용 방법을 안내하는 가이드 링크와 본사에 문의할 수 있는 창구를 제공한다.

---

## 3. 세부 개발 명세

이 페이지의 모든 항목은 점주(가맹점주 또는 운영자)에게 **읽기 전용(Read-Only)**으로 노출되며 내용 변경이 불가능하다. 수정이 필요한 경우 본사 문의를 통해서만 가능하다.

### 3.1. 프론트엔드 (노출 항목)

| 분류 | 기능 / 표기 항목명 | 노출 데이터 형태 | 비고 |
| :--- | :--- | :--- | :--- |
| **브랜드 정보** | 슈퍼어드민에서 세팅한 브랜드 정보 | Text / Image | 서비스명, 브랜드 로고, 기본 사업자 정보 등 표기 |
| **운영 유형** | 브랜드의 주문유형 | Badge / Tags | 배달, 포장, 예약, 이쿠폰 등 활성화된 유형 표시 |
| | 초기 세팅 운영시간 | Text | 요일별 초기 설정 시간 (ex: 평일 09:00~22:00) |
| | 초기 세팅 배달비 | Text (Currency) | 기본 배달비 표기 |
| | 초기 세팅 최소 주문금액 | Text (Currency) | 배달 및 포장에 대한 최소 주문 금액 |
| **연동 및 계약** | 계약 PG사 | Text + Badge | PG사 명칭 및 연동주체(플랫폼 / 직접계약) |
| | 연동 포스 (POS) | Text + Badge | POS사 명칭 및 연동주체(플랫폼 / 직접계약) |
| | 계약기간 | Text (Date range) | YYYY.MM.DD ~ YYYY.MM.DD |
| | 운영수수료 | Text (%) | 수수료율 N% |
| | 운영 요금제 | Text | 적용중인 요금제 등급 명칭 (ex: 요금제 A) |
| | 담당자 연락처 | Text | 본사 시스템 설정 담당자 번호 및 내선 |
| **지원 채널** | 기타 사이트 사용 가이드 (기능사용방법) | External Link / Modal | 별도의 가이드 페이지(노션 등)나 FAQ 팝업 버튼 |
| | 문의 (본사) | Button / Link | 1:1 본사 카톡 문의, 이메일 전송, 또는 내부 문의 게시판 링크 |

### 3.2. 백엔드

모든 시스템 환경설정값은 단일 API를 통해 읽어온다.

#### API 엔드포인트

| Method | Path | 설명 |
| :--- | :--- | :--- |
| GET | `/api/settings/brand-config` | 읽기 전용으로 제공되는 위 12가지 설정 데이터의 단건 통합 반환 |

**[비즈니스 로직 제약사항]**
- 사용자(점주/어드민) 요청에 의한 `UPDATE/PUT` 등 설정값 수정 API 엔드포인트는 어드민 대시보드 백엔드 권한에 존재하지 않는다. 모두 수동(수퍼어드민 전용 포털 또는 DB 직권) 처리로 간주된다.
- 노출되는 결제 수수료나 요금제는 인증된 계정별로 다르게 반영되는 경우가 있으므로 로그인 세션 기반 데이터 조회를 보장해야 한다.

## 4. 정상작동 시나리오

### 시나리오 1: 설정 페이지 데이터 조회 및 가이드 이동

| 단계 | 사용자 행동 | 시스템 응답 | 검증 포인트 |
| :---: | :--- | :--- | :--- |
| 1 | 사이드바 '설정' 메뉴 클릭 | `GET /api/settings/brand-config` 호출 | 페이지 진입과 동시에 로딩 시작 |
| 2 | 데이터 노출 | 각 항목들이 Disabled된 Input 또는 순수 Text 형태로 렌더링 | 수정, 저장 버튼이 화면에 전면 존재하지 않음 |
| 3 | [문의 (본사)] 버튼 클릭 | 본사 카카오톡 채널/고객센터 창 새 탭으로 오버레이 | 지원 채널 정상 연동 확인 |
| 4 | [기능사용방법] 클릭 | 사이트 매뉴얼 페이지 이동 | 가이드 문서 또는 모달 정상 노출 확인 |
