## 1. 수집 항목 분류 및 동의 요건

### 1.1 필수 수집 항목 (서비스 이용약관 동의)

| 항목 | 타입 필드 | 목적 | 보관기간 |
|------|-----------|------|----------|
| 이름 | `Member.name` | 본인 식별 | 탈퇴 후 5일 이내 파기 |
| 전화번호 | `Member.phone` | 본인 인증, 주문 연락 | 탈퇴 후 5일 이내 파기 |
| 회원 아이디 | `Member.memberId` | 로그인 | 탈퇴 후 5일 이내 파기 |

### 1.2 선택 수집 항목 (별도 동의)

| 항목 | 타입 필드 | 목적 | 동의 유형 | 보관기간 |
|------|-----------|------|-----------|----------|
| 이메일 | `Member.email` | 알림, 영수증 발송 | privacy | 탈퇴 시 파기 |
| 생년월일 | `Member.birthDate` | 생일 쿠폰, 연령 확인 | privacy | 탈퇴 시 파기 |
| 성별 | `Member.gender` | 맞춤 추천 | privacy | 탈퇴 시 파기 |
| 배달지 주소 | `DeliveryAddress` | 배달 주문 | privacy | 탈퇴 시 파기, 미사용 1년 자동삭제 |
| 위치정보 | `DeliveryAddress.lat/lng` | 배달 거리 계산 | location | 목적 달성 시 즉시 파기 |

### 1.3 마케팅 동의 항목 (별도 opt-in)

| 항목 | 타입 필드 | 동의 유형 |
|------|-----------|-----------|
| 마케팅 수신 동의 | `Member.marketingAgreed` | marketing |
| 푸시 알림 | `Member.pushEnabled` | marketing |

| 이메일 수신 | `Member.emailEnabled` | marketing |

### 1.4 제3자 제공 항목

| 항목 | 제공 대상 | 동의 유형 |
|------|-----------|-----------|
| SNS 연동 키 | `SnsConnection.snsKey` → 카카오/네이버/구글/애플 | third_party |
| 주문 정보 | 가맹점 → 주문 처리 | third_party |

## 2. 동의 체계 (TermsAgreement)

### 2.1 현행 약관 유형

```typescript
type TermsType = 'service' | 'privacy' | 'marketing' | 'location' | 'third_party';
```

### 2.2 법적 요구사항 대비 점검

| 요구사항 | 현행 상태 | 조치 필요 |
|----------|-----------|-----------|
| 필수/선택 동의 분리 | TermsType으로 분리됨 | ✅ 충족 |
| 동의 시점 기록 | `agreedAt: Date` | ✅ 충족 |
| 동의 버전 관리 | `version: string` | ✅ 충족 |
| 동의 철회 기능 | 미구현 | ❌ 추가 필요 |
| 동의 철회 시점 기록 | 미구현 | ❌ 추가 필요 |
| 만 14세 미만 법정대리인 동의 | 미구현 | ⚠️ 외식앱 특성상 저위험, 단 확인 로직 권장 |
| 마케팅 동의 야간 발송 제한 (21~08시) | 미확인 | ❌ 캠페인 발송 로직에 시간 체크 필요 |

### 2.3 추가 필요 필드

```typescript
// TermsAgreement 확장
interface TermsAgreement {
  termsType: TermsType;
  agreedAt: Date;
  version: string;
  revokedAt?: Date | null;     // 동의 철회 시점 (추가 필요)
  revokeReason?: string;       // 철회 사유 (선택)
}
```

## 3. 안전성 확보조치 (기술적 보호)

### 3.1 암호화 기준 (안전성 확보조치 기준 제7조)

| 대상 | 저장 시 | 전송 시 | 현행 |
|------|---------|---------|------|
| 비밀번호 | 일방향 해시 (SHA-256+salt 이상) | TLS 1.2+ | 백엔드 구현 필요 |
| 전화번호 | AES-256 암호화 저장 | TLS 1.2+ | `phone` 마스킹만 적용 → 서버 암호화 필요 |
| 이메일 | AES-256 암호화 저장 | TLS 1.2+ | 서버 암호화 필요 |
| 생년월일 | AES-256 암호화 저장 | TLS 1.2+ | 서버 암호화 필요 |
| 배달 주소 | AES-256 암호화 저장 | TLS 1.2+ | 서버 암호화 필요 |
| SNS 연동 키 | AES-256 암호화 저장 | TLS 1.2+ | 서버 암호화 필요 |

### 3.2 접근통제 (안전성 확보조치 기준 제6조)

| 조치 | 설명 | 프론트엔드 반영 |
|------|------|-----------------|
| 역할 기반 접근 | 회원 개인정보 열람 → 관리자 이상 권한 | Staff 권한 체계로 구현 |
| 마스킹 표시 | 전화번호 `010-****-1234`, 이메일 `a***@email.com` | 리스트에서 기본 마스킹 |
| 마스킹 해제 | 상세 보기 시 "개인정보 열람" 버튼 + 사유 입력 | ❌ 추가 필요 |
| 열람 로그 | 누가 언제 어떤 회원 정보를 봤는지 기록 | ❌ API 로그 필요 |
| 다운로드 제한 | 회원 목록 내보내기 → 권한 체크 + 사유 기록 | `MemberExportRequest`에 사유 필드 추가 필요 |

### 3.3 접속기록 보관 (안전성 확보조치 기준 제8조)

| 규모 | 보관기간 |
|------|----------|
| 5만명 미만 | 1년 이상 |
| 5만명 이상 또는 민감정보 처리 | 2년 이상 |

## 4. 보관 및 파기 정책

### 4.1 법정 보관기간

| 데이터 | 근거 법령 | 보관기간 |
|--------|-----------|----------|
| 주문/결제 기록 | 전자상거래법 | 5년 |
| 소비자 불만/분쟁 처리 기록 | 전자상거래법 | 3년 |
| 로그인/접속 기록 | 통신비밀보호법 | 3개월 |
| 표시/광고 기록 | 전자상거래법 | 6개월 |
| 계약/청약철회 기록 | 전자상거래법 | 5년 |

### 4.2 파기 절차

| 상황 | 조치 | 기한 |
|------|------|------|
| 회원 탈퇴 | 개인식별정보 즉시 파기, 법정보관 항목 분리보관 | 5일 이내 |
| 1년 미이용 (휴면) | 개인정보 분리 보관 또는 파기 | `Member.status = 'dormant'` |
| 휴면 후 추가 미이용 | 완전 파기 | 분리보관 후 별도 고지 |
| 보관기간 경과 | 복구 불가능한 방법으로 파기 | 지체 없이 |
| 배달지 주소 미사용 | 자동 삭제 | `lastUsedAt` 기준 1년 |

### 4.3 파기 방법

- DB 레코드: 물리적 삭제 (DELETE) + 백업 파일 포함
- 파일: 안전한 덮어쓰기 또는 디가우징
- 암호화 저장분: 암호키 폐기로 복원 불가 처리
