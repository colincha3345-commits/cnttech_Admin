# 시스템 구현 기능 정의 명세 (Implemented Features)

본 문서는 관리자 대시보드 및 시스템에 현재까지 설계되고 구현된 핵심 도메인별 기능 리스트와, 각 기능의 속성, 프로세스 플로우, 필수 작동 요소를 정리한 문서입니다.

---

## 1. 회원 (Member)
회원의 기본 정보 및 소셜 연동, 동의 이력, 등급 등을 관리합니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **기본 정보:** 회원ID, 이름, 연락처(마스킹), 이메일, 성별, 생년월일 <br/>- **상태 및 등급:** 활성/휴면/장기미접속/탈퇴, VIP/Gold/Silver/Normal 등급 <br/>- **인증 및 동의:** 소셜 연동 정보(카카오, 네이버, 애플 등), 약관 동의 이력(서비스, 마케팅, 위치 등) <br/>- **활동 내역:** 포인트 잔액, 총 주문 금액, 주문 횟수, 마지막 접속일 |
| **운영 플로우 (Flow)** | 1. **회원가입/로그인:** 소셜 연동 또는 일반 가입을 통한 계정 생성 <br/>2. **활동 및 적립:** 상품 구매 및 프로모션 참여 시 포인트 누적, 누적 조건에 따른 등급 갱신 <br/>3. **상태 전환:** 장기 미접속 시 휴면 처리, 요청 시 탈퇴 처리 |
| **필수 작동 요소** | - 소셜 로그인 연동(OAuth) 식별자 매핑 로직 <br/>- 마케팅, 앱 푸시, SMS, 이메일 수신 동의 분리 구조 <br/>- 실시간 포인트 적립/차감 이력 트래킹 로직 |

---

## 2. 매장 / 스토어 (Store)
가맹점 및 본사 직영점의 영업 상태와 주문 접수 환경을 관리합니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **기본/계약 정보:** 사업자 번호, 대표자명, 주소(지역 기반), 점주 연락처, 정산 계좌, 계약 상태/기한 <br/>- **영업시간:** 실시간 앱 노출 상태(영업중/준비중/휴식/종료/임시휴업), 평일/주말 및 요일별 개별 영업 및 휴게(Break-time) 시간 <br/>- **휴무 정책:** 정기휴무(매주/매월N주차/지정일), 비정기 휴무 <br/>- **주문/배달 설정:** 배달/포장 이용 가능 여부, 최소 주문 금액, 배달비 정책(기본/거리별/금액별), 예약 가능 여부 설정 |
| **운영 플로우 (Flow)** | 1. **가맹점 등록/계약:** 점주 정보 및 계약 정보 입력 후 활성화 대기 <br/>2. **운영 세팅:** 요일별 영업시간 및 휴무일 캘린더 세팅, 배달 권역 및 배달비 할증 정보 세팅 <br/>3. **실시간 제어:** 매장 사정에 따른 "본 앱 상의 실시간 영업 상태(준비중, 휴업 등)" 즉각 토글 <br/>4. **주문 접수:** 예약 및 즉시 주문 건 접수 및 처리 |
| **필수 작동 요소** | - 앱 영업 상태의 실시간 동기화 <br/>- 현재 시간 기반 예약 기능 활성화 조건(현재시간 + N분 후 예약 가능) 제어 <br/>- 복합 정기휴무 사이클(월 N번째 요일 등) 자동 계산 엔진 |

## 3. 할인 (Discount)
정시적, 상시적으로 상품 단위 도는 총액에 적용되는 할인 정책을 정의합니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **할인 유형/방식:** 자사할인 / 증정할인 (상품 N+1, 특정상품 구매시, 최소금액 충족), 정액/정률(%) 방식 <br/>- **타겟 및 시간:** 전체/카테고리/특정상품 대상, 상시/특정기간/요일 및 시간대(Schedule) 적용 <br/>- **조건/비용:** 최소 주문 금액 한도, 정률 할인 시 최대 할인금액 상한선, 정산 비율(본사 vs 가맹점) <br/>- **금액 단위 설정:** 절사/반올림 방식 (1원/10원/100원 단위) |
| **운영 플로우 (Flow)** | 1. **정책 생성:** 할인율, 증정 사은품, 정산 비율 등의 규칙 정의 <br/>2. **기간/타겟 맵핑:** 노출 채널(앱/PC/모바일웹), 조건(배달/포장), 타겟 상품 매핑 <br/>3. **적용 및 정산:** 고객 장바구니에서 조건 부합 시 자동/수동 적용, 관리자에서는 매월 가맹점과 비율에 맞춘 정산 데이터 집계 |
| **필수 작동 요소** | - 할인 반올림/절사 규칙(RoundingSetting) 적용기 <br/>- 주문 상품 기반의 실시간 N+1 및 증정품 조건 충족 계산기 <br/>- 다중 매장 일괄 적용 및 제외 필터링 |

---

## 4. 쿠폰 (Coupon)
고객이 직접 획득하고 주문 시 선택하여 할인 받는 지불 수단 역할을 합니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **적용 범위:** 장바구니 총 금액 / 특정 상품 / 배달비 전용 할인 <br/>- **금액 및 조건:** 정액/정률 할인값, 최소 주문 금액 제한 <br/>- **사용 제한:** 특정 요일/시간대 사용 제한 스케줄, 가맹점 인클루드/익스클루드, 1인 1회 제한유무 <br/>- **발행 정보:** 총 발행 가능 수량, 사용된 수량, 유효기간 옵션 |
| **운영 플로우 (Flow)** | 1. **쿠폰 생상:** 적용 대상, 제약 조건, 발급 총량 설정 후 배포 <br/>2. **지급 대기:** 혜택 캠페인을 통한 자동 지급 모델 혹은 회원이 이벤트 페이지에서 다운로드 <br/>3. **사용 및 소멸:** 유효기간 또는 발급수량 매진 시 상태(Exhausted/Expired) 변경, 주문 완료 시 사용 처리 |
| **필수 작동 요소** | - 결제 전 쿠폰 유효기간 및 사용 요일/스케줄(TimeRange) 실시간 검증 로직 <br/>- 쿠폰 잔여 수량 동시성 제어 처리 |

---

## 5. 혜택 캠페인 (Benefit Campaign)
특정 고객 행동이나 지정된 조건에 따라 쿠폰이나 포인트를 "자동/일괄 지급"하는 자동화 마케팅 기능입니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **트리거 (조건):** 회원가입, 멤버십 등급 달성, 생일, 특정 조건 주문(N번째 주문 등), 추천인, 관리자 수기 업로드(CSV), 난수 쿠폰코드 매칭 <br/>- **혜택 페이로드:** 지급할 특정 쿠폰 ID 맵핑, 특정 적립 포인트 한도 및 유효기간 매핑 <br/>- **지연 옵션:** 조건 충족 즉시 발급, N시간 후 발급, N일 후 대기 발급 설정 |
| **운영 플로우 (Flow)** | 1. **트리거 생성:** 시스템 내 특정 이벤트(가입, 생일)와 혜택을 매칭시켜 캠페인을 진행(Active) 상태로 변경 <br/>2. **조건 감지:** 회원이 물건을 구매하거나 회원 등급 배치 처리가 돌 시 시스템이 조건 부합 여부 식별 <br/>3. **보상 지급:** 지연 스케줄에 따라 사용자 쿠폰함 및 포인트 함으로 비동기 발급 완료 |
| **필수 작동 요소** | - 프로모션 난수 코드 (PromoCode) 일괄 생성 및 외부 CSV 업로드/파싱 툴 <br/>- 비동기 지연 지급(Delay) 작업 처리 스케줄러 기능 |

## 6. 이벤트 (Event)
프로모션 콘텐츠를 노출하고 딥링크 이동, SNS 공유 및 참여자(개인정보)를 수집하는 페이지/배너 기능입니다.

| 분류 | 세부 내용 |
| :--- | :--- |
| **핵심 속성 (Properties)** | - **게시 정보:** 이벤트 유형(일반/참여형), 썸네일/상세 배너, 게시 예약시간, 노출 상태 <br/>- **동선(Link) 기능:** 앱 딥링크, 프로모션 주문 연결 버튼, SNS 공유(카카오/인스타 등) 텍스트 및 대표 이미지 메타 설정 <br/>- **참여자 수집 폼:** 연락처/이메일/주소 등 동적 폼 필드 활성화 여부, 개인정보 및 제3자 제공 동의 약관 내용 <br/>- **통계 자료:** 상세 조회수, 주문버튼 클릭수, 공유수, 참여자 전환율(Conversion Rate) |
| **운영 플로우 (Flow)** | 1. **이벤트 기획:** 상세 이미지 리소스 업로드 빛 예약 발행일정 지정 <br/>2. **동의 서식 지정 (참여형인 경우):** 경품 발송을 위해 요구되는 수집 필드 및 동의 문구 명시 <br/>3. **노출 및 바이럴:** 메인 화면 및 팝업을 통해 사용자 인입, SNS 카카오 공유 등을 통한 외부 유입 유도 <br/>4. **마감 및 통계 분석:** 게재 기간 만료 후 성과 모니터링 메뉴에서 참여자 정보 CSV 다운로드 및 성과 분석 |
| **필수 작동 요소** | - 외부 공유 시 OG(Open Graph) 태그 및 앱 진입 딥링크 매핑 <br/>- 수집된 개인정보 보안 암호화 보관 및 조회 (CollectionMode) |
