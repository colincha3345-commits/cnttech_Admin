# 할인/쿠폰 도메인 정책서

본 문서는 관리자 대시보드의 **할인(Discount)**, **쿠폰(Coupon)**, **캠페인(Campaign)**, **포인트(Point)** 기능에 적용되는 비즈니스 정책을 정의한다.
프론트엔드 UI 분기, 백엔드 유효성 검증, 배치 스케줄러에서 공통으로 참조한다.

---

## 0. 용어 정의

### 0.1. 할인 (Discount) — 할인 방식별

| 한글 용어 | 영문 키 | 타입/필드 | 정의 |
| :--- | :--- | :--- | :--- |

| **증정할인** | `gift` | `DiscountType` | 조건 충족 시 상품을 무료 증정 (N+1, 최소금액 등) |
| **정률 할인** | `percentage` | `DiscountMethod` | 주문금액의 N% 할인 |

| **할인 대상: 전체** | `all` | `DiscountTargetType` | 전체 상품에 할인 적용 |
| **할인 대상: 카테고리** | `category` | `DiscountTargetType` | 특정 카테고리 상품에만 적용 |

| **올림** | `ceil` | `RoundingType` | 할인 금액 올림 |
| **내림(버림)** | `floor` | `RoundingType` | 할인 금액 내림 |
| **상시 할인** | `always` | `DiscountPeriodType` | 기간 제한 없이 상시 적용 |
| **기간 할인** | `period` | `DiscountPeriodType` | 시작일~종료일 한정 |
| **증정 조건: 상품 구매** | `product_purchase` | `GiftConditionType` | 특정 상품 구매 시 증정 |

| **특정 상품 적용** | `specific_product` | `CouponApplyScope` | 선택한 상품에만 할인 |
| **배달비 적용** | `delivery_fee` | `CouponApplyScope` | 배달비에만 할인 |
| **채널: 전체** | `all` | `CouponChannel` | 전체 채널 사용 가능 |
| **채널: APP** | `app` | `CouponChannel` | 앱에서만 사용 가능 |
| **채널: PC웹** | `pc_web` | `CouponChannel` | PC 웹에서만 사용 가능 |
| **채널: 모바일웹** | `mobile_web` | `CouponChannel` | 모바일 웹에서만 사용 가능 |

## 1. 할인 중복 적용 정책

### 1.1. 핵심 원칙: 모든 할인은 상호 배타적

**할인(Discount), 쿠폰(Coupon), 포인트(Point)**는 모두 중복 적용 불가하다. 장바구니에 할인 상품이 1개라도 포함되면 쿠폰/포인트를 사용할 수 없다.

| 조합 | 허용 여부 | 비고 |
| :--- | :---: | :--- |
| 할인 A + 할인 B | ✅ 허용 | 본사할인/증정할인 동일 유형 내 복수 가능 |
| 할인 A + 쿠폰 A | ❌ 불가 | 할인상품이 장바구니에 있으면 쿠폰 차단 |
| 쿠폰 A + 쿠폰 B | ❌ 불가 | 쿠폰은 최대 1건 |
| 할인 A + 포인트 | ❌ 불가 | 할인 적용 시 포인트 차단 |
| 쿠폰 A + 포인트 | ❌ 불가 | 쿠폰 적용 시 포인트 차단 |
| 정가 상품만 + 쿠폰 A | ✅ 허용 | 할인 없는 상품만 있을 때 쿠폰 가능 |
| 포인트 단독 사용 | ✅ 허용 | 할인/쿠폰 없을 때만 |

### 1.2. 프론트엔드 적용 규칙

| 시점 | UI 동작 | 상세 |
| :--- | :--- | :--- |
| 할인 상품을 장바구니에 추가 시 | 쿠폰 선택 영역 비활성화 | "할인 상품이 포함되어 쿠폰을 사용할 수 없습니다" 안내 |
| 쿠폰 적용 상태에서 할인 상품 추가 시 | 쿠폰 자동 해제 + Toast | "할인 상품이 추가되어 쿠폰이 해제되었습니다" |
| 할인 또는 쿠폰 적용 상태에서 포인트 사용 시도 | 포인트 입력 비활성화 | "할인/쿠폰 적용 중에는 포인트를 사용할 수 없습니다" |
| 포인트 사용 중 할인 상품 추가 또는 쿠폰 적용 시 | 포인트 자동 해제 + Toast | "할인이 적용되어 포인트가 해제되었습니다" |
| 주문 금액 변경으로 최소 주문금액 미달 시 | 쿠폰 자동 해제 + 안내 | "주문 금액이 변경되어 쿠폰이 해제되었습니다" |

### 1.3. 결제창 쿠폰 선택 리스트

결제(주문) 화면에서 "쿠폰 적용" 영역에 노출되는 쿠폰 목록이다. **현재 주문 맥락(장바구니 상품, 주문금액, 채널, 매장, 시간)에 따라 사용 가능 여부를 실시간 판단**한다.

| 상태 | 표시 | UI |
| :--- | :--- | :--- |
| 사용 가능 | 정상 노출 + 선택 가능 | 쿠폰 카드 활성, 할인 금액 미리보기 |
| 할인상품 포함으로 사용 불가 | 노출 + 비활성(gray) | "할인 상품이 포함되어 사용할 수 없습니다" |
| 최소 주문금액 미달 | 노출 + 비활성(gray) | "최소 주문금액 N원 이상 시 사용 가능" |
| 적용 대상 상품 없음 | 노출 + 비활성(gray) | "적용 가능한 상품이 장바구니에 없습니다" |
| 채널/주문유형 불일치 | 노출 + 비활성(gray) | "현재 주문 유형에서 사용할 수 없습니다" |
| 요일/시간 제한 밖 | 노출 + 비활성(gray) | "사용 가능 시간이 아닙니다" |
| 매장 제한 불일치 | 노출 + 비활성(gray) | "해당 매장에서 사용할 수 없습니다" |
| 쿠폰 정지/만료 | **비노출** | 결제창 리스트에서 제거 |

> 결제창에서는 **사용 가능한 쿠폰이 최상단**, 비활성 쿠폰이 하단에 정렬된다. 비활성 쿠폰에는 사유 텍스트가 필수 표시된다.

### 1.4. 마이페이지 쿠폰함

마이페이지 > 쿠폰함은 **고객이 보유한 전체 쿠폰 현황을 확인하는 화면**이다. 주문 맥락과 무관하게 보유 쿠폰을 표시한다.

| 상태 | 표시 | UI |
| :--- | :--- | :--- |
| 사용 가능 (유효기간 내) | 정상 노출 | 쿠폰 카드 + 잔여일수 "D-N" 표시 |
| 사용 완료 | 노출 + "사용완료" 표시 | 회색 처리, 사용일자/주문번호 표시 |
| 기간 만료 | 노출 + "기간만료" 표시 | 회색 처리, 만료일 표시. 만료 후 30일 경과 시 비노출 |
| 쿠폰 정지 (유예기간 중) | 정상 노출 + 사용 가능 | 유예 만료일까지 사용 가능 안내 |
| 쿠폰 정지 (유예 만료 후) | **비노출** | 쿠폰함에서 제거 |
| 관리자 삭제 | **비노출** | 쿠폰함에서 즉시 제거, 별도 알림 없음 |

> 마이페이지 쿠폰함은 고객의 쿠폰 보유 현황 파악이 목적이므로, "사용완료"/"기간만료" 쿠폰도 일정 기간 노출하여 "쿠폰이 사라졌다"는 CS 문의를 방지한다.

### 1.5. 백엔드 검증 규칙

| 검증 항목 | 조건 | 에러 코드 |
| :--- | :--- | :--- |
| 할인+쿠폰 동시 사용 | 장바구니에 할인상품 포함 + 쿠폰 적용 시도 | `DISCOUNT_COUPON_CONFLICT` |
| 쿠폰 중복 | 요청에 쿠폰 2건 이상 포함 시 | `DUPLICATE_COUPON` |
| 할인+포인트 동시 사용 | 할인/쿠폰 적용 상태에서 포인트 사용 요청 | `DISCOUNT_POINT_CONFLICT` |
| 최소 주문금액 미달 | 쿠폰의 minOrderAmount > 실 결제금액 | `MIN_ORDER_AMOUNT_NOT_MET` |
| 적용 대상 불일치 | 쿠폰의 applicableProductIds에 주문 상품 미포함 | `PRODUCT_NOT_APPLICABLE` |
| 사용 채널 불일치 | 쿠폰의 channel과 주문 채널 불일치 | `CHANNEL_NOT_APPLICABLE` |
| 주문 유형 불일치 | 쿠폰의 orderType과 주문 유형 불일치 | `ORDER_TYPE_NOT_APPLICABLE` |
| 가맹점 제한 | storeRestriction에 해당 매장 미포함 | `STORE_NOT_APPLICABLE` |
| 스케줄 제한 | 현재 시각이 availableDays/availableTimeRanges 밖 | `SCHEDULE_NOT_APPLICABLE` |

### 1.6. 할인 적용 순서

복수의 할인이 존재할 경우 **할인 금액이 가장 큰 것부터** 적용 (고객 혜택 최대화).

| 적용 순서 | 기준 | 설명 |
| :---: | :--- | :--- |
| 1순위 | 할인 금액 최대 | 고객에게 가장 유리한 할인을 우선 적용 |
| 2순위 | 생성일 오래된 순 | 동일 금액 시 먼저 생성된 할인 적용 |

## 1-A. 쿠폰 용어 정의

| 용어 | 정의 | 필드명 |
| :--- | :--- | :--- |
| 발행 수 | 관리자가 설정한 총 쿠폰 수량 | `totalCount` |
| 소진 | 고객에게 발급 완료된 쿠폰 수 (다운로드/난수 코드 입력 포함) | `issuedCount` |
| 쿠폰사용 | 실제 주문에서 사용 완료된 쿠폰 수 | `usedCount` |
| 잔여 수 | 발행 수 − 소진 수 = 아직 발급 가능한 수량 | `totalCount - issuedCount` |

> "소진"과 "쿠폰사용"은 반드시 구분한다. 소진은 발급 시점, 쿠폰사용은 결제 완료 시점에 카운트.

## 2. 쿠폰 생성 시 할인 가능 상품 변경 정책

### 2.1. 시나리오

쿠폰 생성 시 `applicableProductIds`에 상품 A를 등록했으나, 이후 상품 A의 상태가 `soldout` 또는 할인 불가로 변경된 경우.

### 2.2. 정책 규칙

**핵심 원칙: 쿠폰 적용 대상이 아닌 상품은 적용 상품 목록에서 즉시 제외한다. 별도 표시하지 않는다.**

| 상황 | 처리 방식 | 프론트엔드 UI |
| :--- | :--- | :--- |
| 쿠폰 대상 상품이 `soldout`/삭제/속성 변경 | `applicableProductIds`에서 해당 상품 ID 자동 제외 | 적용 상품 목록에서 즉시 제거 (별도 표시 없음) |
| 쿠폰 대상 상품 전체가 비가용 | 쿠폰 상태를 `suspended`(정지)로 자동 전환 | 관리자에게 "쿠폰 정지됨" 알림 Toast + 목록에 경고 Badge |
| 정지된 쿠폰의 대상 상품이 다시 활성화 | `suspended` → `active` 자동 복원 불가. 관리자 수동 재활성화 필요 | 관리자가 직접 "활성화" 버튼 클릭 필요 |

### 2.3. 백엔드 처리 흐름

| 단계 | 주체 | 처리 |
| :--- | :--- | :--- |
| 1. 상품 상태 변경 이벤트 | 상품 서비스 | 상품 상태 변경 시 `product.status.changed` 이벤트 발행 |
| 2. 쿠폰 영향 조회 | 쿠폰 서비스 | `applicableProductIds`에 해당 상품을 포함하는 활성 쿠폰 조회 |
| 3. 적용 상품 목록 갱신 | 쿠폰 서비스 | 비가용 상품 ID를 `applicableProductIds`에서 제거 |
| 4. 전체 상품 비가용 판정 | 쿠폰 서비스 | 모든 적용 대상 상품이 제거됨 → `suspended` 전환 |
| 5. 관리자 알림 | 알림 서비스 | 정지된 쿠폰 건수 + 쿠폰명을 관리자 대시보드에 알림 |

### 2.4. 프론트엔드 관리자 화면

| 화면 | 표시 내용 |
| :--- | :--- |
| 쿠폰 목록 | `suspended` 상태 쿠폰에 주황색 Badge("정지") + 사유 툴팁("대상 상품 전체 품절") 표시 |
| 쿠폰 상세 (적용 상품 섹션) | 유효한 상품만 표시. 비가용 상품은 목록에서 자동 제거됨 |
| 쿠폰 수정 | `ProductSelector`가 유효 상품 ID만 필터링하여 표시. 적용 상품이 1개 이상이면 `active`로 전환 가능 |

### 2.5. 고객 앱 — 상품 상태 변경 시 쿠폰 처리

고객 앱에서 쿠폰 대상 상품의 상태가 변경(품절/삭제)되었을 때의 처리 정책이다.
마이페이지 쿠폰함의 전체 표시 정책은 §1.4를, 결제창 쿠폰 리스트 정책은 §1.3을 참고한다.

| 상황 | 마이페이지 쿠폰함 | 결제창 쿠폰 리스트 |
| :--- | :--- | :--- |
| 대상 상품 일부 품절/삭제 | **UI 변경 없음**. 쿠폰 노출 유지 | 적용 가능 상품이 없으면 비활성 처리 |
| 품절 상품으로 쿠폰 적용 시도 | — | 서버 거부 → "선택한 상품에 쿠폰을 적용할 수 없습니다" Toast |
| 쿠폰이 `suspended` (전체 상품 비가용) | 비노출 (§1.4 참조) | 비노출 (§1.3 참조) |
| `suspended` → 관리자 재활성화 시 | 재노출 | 재노출 |

## 3. 쿠폰 삭제 정책

### 3.1. 핵심 원칙

쿠폰은 **유효기간 만료 후 7일(유예기간)이 경과한 이후에만** 관리자 리스트에서 삭제할 수 있다.
유예기간 중에는 삭제 버튼이 비활성화된다.

### 3.2. 쿠폰 생명주기

| 단계 | 상태 | 삭제 가능 | 고객 쿠폰함 | 비고 |
| :---: | :--- | :---: | :--- | :--- |
| 1 | `active` (활성) | ❌ | 노출 + 사용 가능 | 정상 운영 상태 |
| 2 | `inactive` (비활성) | ❌ | 비노출 | 관리자가 수동 비활성화 |
| 3 | `suspended` (정지) | ✅ | 유예기간 중 사용 가능, 유예 후 비노출 | 관리자 정지. 유예기간(n일) 부여 |
| 4 | `exhausted` (소진) | ❌ | 노출 + "소진" 표시 | 총 발행 수량 소진 |
| 5 | `expired` (만료) | ✅ | 비노출 | 유효기간(endDate) 경과 |
| 6 | 자동 삭제 대기 | - | 비노출 | 정지/만료 후 7일 경과 시 자동 삭제 |

### 3.2.1. 정지(사용정지) 정책

| 항목 | 설명 |
| :--- | :--- |
| 정지 트리거 | 관리자가 쿠폰 목록에서 "정지" 버튼 클릭 |
| 유예기간 | 0~30일 설정 가능 (기본 7일). 유예기간 동안 기발급 쿠폰은 계속 사용 가능 |
| 유예 만료 후 | 고객 쿠폰함에서 비노출 + 사용 불가 |
| 자동 삭제 | 유예 만료일 + 7일 후 쿠폰 자동 삭제 (고객 쿠폰도 함께 삭제) |
| `autoDeleteAt` | `suspendedAt + gracePeriodDays + 7일`로 자동 계산 |
| 재활성화 | 자동 삭제 전까지 관리자가 수동으로 재활성화 가능 → `autoDeleteAt` 초기화 |

### 3.3. 삭제 가능 조건 (AND 조건)

| 조건 | 설명 |
| :--- | :--- |
| 쿠폰 상태가 `expired` | 유효기간이 만료된 상태여야 한다 |
| `endDate + 7일 < 현재 시각` | 유예기간(7일)이 경과해야 한다 |
| 미정산 주문 없음 | 해당 쿠폰을 사용한 미정산(pending) 주문이 0건이어야 한다 |

### 3.4. 프론트엔드 삭제 UI

| 화면 | UI 동작 |
| :--- | :--- |
| 쿠폰 목록 - `expired` + 유예기간 미경과 | 삭제 버튼 비활성(disabled) + 툴팁 "만료 후 7일 경과 시 삭제 가능 (삭제 가능일: YYYY-MM-DD)" |
| 쿠폰 목록 - `expired` + 유예기간 경과 | 삭제 버튼 활성. 클릭 시 확인 모달 표시 |
| 삭제 확인 모달 | "쿠폰을 삭제하면 고객 쿠폰함에서도 즉시 제거됩니다. 이 작업은 되돌릴 수 없습니다. 삭제하시겠습니까?" |
| 미정산 주문 존재 시 | 삭제 불가 + 안내 "해당 쿠폰을 사용한 미정산 주문이 N건 있습니다. 정산 완료 후 삭제할 수 있습니다." |

### 3.5. 삭제 시 처리 범위

| 대상 | 처리 |
| :--- | :--- |
| 관리자 쿠폰 목록 | 쿠폰 레코드 소프트 삭제 (`deletedAt` 타임스탬프 기록) |
| 고객 쿠폰함 (MemberCoupon) | 해당 쿠폰 ID의 모든 발급 레코드를 비노출 처리 (`isDeleted = true`) |
| 사용 완료된 기록 | 변경 없음. 주문 내역의 쿠폰 사용 기록은 영구 보존 |
| 정산 데이터 | 변경 없음. 이미 정산된 쿠폰 할인 금액은 보존 |
| 감사 로그 | `COUPON_DELETED` 액션 기록 (삭제자, 쿠폰ID, 쿠폰명, 발급수/사용수) |

### 3.6. 배치 스케줄러 (자동 만료 처리)

| 스케줄 | 처리 | 비고 |
| :--- | :--- | :--- |
| 매일 00:00 | `endDate < 현재` 이고 `status = active/inactive` → `expired` 전환 | 만료 자동 전환 |
| 매일 00:00 | `autoDeleteAt < 현재` → 소프트 삭제 실행 | 정지/만료 쿠폰 자동 삭제 (유예+7일 경과) |
| 매일 00:00 | `autoDelete = true` 이고 `endDate + 7일 < 현재` → 소프트 삭제 실행 | 만료 시 자동 삭제 (autoDelete 옵션 활성 시) |
| 삭제 실행 시 | 고객 쿠폰함 일괄 비노출 + 감사 로그 기록 | 배치 삭제도 감사 로그 필수 |

### 3.7. 고객 앱 쿠폰함 삭제 처리

| 상황 | 고객 앱 UI |
| :--- | :--- |
| 쿠폰 만료 | 쿠폰함에서 비노출 (만료 즉시) |
| 관리자 삭제 | 쿠폰함에서 즉시 비노출. 별도 알림 없음 |
| 삭제된 쿠폰으로 주문 시도 | "사용할 수 없는 쿠폰입니다" 에러 반환 |
| 쿠폰 사용 완료 후 삭제 | 주문 내역에서 "쿠폰 할인 N원" 표시는 유지 (쿠폰명은 스냅샷) |

## 5. 정산 연동 정책 (할인/쿠폰 부담 비율)

| 항목 | 설명 |
| :--- | :--- |
| 할인 정산 비율 | `headquartersRatio` + `franchiseRatio` = 100(%). 할인별로 개별 설정 |
| 쿠폰 정산 비율 | `settlementRatio.headquartersRatio` + `settlementRatio.franchiseRatio` = 100(%) |
| 정산 반영 시점 | 주문 완료(completed) 시점에 쿠폰/할인 사용분을 정산 데이터에 반영 |
| 취소 시 정산 | 주문 취소 시 쿠폰/할인 사용분을 정산에서 차감. 이미 정산 완료 시 다음 정산에서 조정 |
