# 관리자 대시보드 및 통계 페이지 개발 문서

## 개요
이 문서는 시스템 운영 현황을 보여주는 메인 대시보드(`Dashboard.tsx`)와 최근 추가된 GA4 트래픽 통계 전용 페이지(`GA4Statistics.tsx`)에 대한 세부 구현 명세입니다. 이 문서는 프론트엔드 개발자가 페이지별 컴포넌트 조합 및 데이터 흐름을 명확하게 파악하고 유지보수할 수 있도록 다나까체로 작성되었습니다.

---

## 1. 페이지 구조 및 라우팅 (Routing)

대시보드 기능은 하위 트리를 가지며 좌측 사이드바(LNB)를 통해 두 가지 뷰로 나뉘어 서비스됩니다. 모두 `read` 권한이 요구되며 모든 인증된 관리자 계정(admin, manager, viewer)이 접근할 수 있습니다.

- **`/dashboard`**: 핵심 운영 지표(매출, 주문, 회원, 문의)를 종합하여 보여주는 통계 요약 뷰입니다.
- **`/dashboard/ga4`**: 유저의 활동 지표(DAU, MAU, 이탈률 등)를 집중적으로 분석하여 시각화한 GA4 통계 현황 뷰입니다.
  - **`/dashboard/ga4/device`**: 디바이스, 브라우저, OS, 해상도 등 다각적인 접속 환경 통계를 제공하는 디바이스 상세 분석 페이지입니다.
  - **`/dashboard/ga4/funnel`**: 퍼널별 일별 추이 및 이탈 분석, TOP 전환 상품 등의 세부 인사이트를 제공하는 퍼널 상세 분석 페이지입니다.

---

## 2. 기능 단위 세부 동작 (Functional Units)

두 페이지 모두 컴포넌트의 통일성과 코드의 재사용성을 높이기 위해 공용 UI 컴포넌트(주로 `StatCard` 및 `InfoCard`)를 활용하여 구성되었습니다.

### 2-1. 메인 대시보드 (`Dashboard.tsx`)
대시보드는 크게 세 가지 구역으로 나뉘어 렌더링됩니다.

1. **상단 요약 (StatCard 4종)** 
   - 전체 주문 금액, 주문 수, 신규 가입 회원 수 등 경영진이 가장 자주 확인하는 핵심 지표를 가로 4열(`admin-grid`)로 배치합니다.
   - `change`(변화율) 속성을 통해 전일 대비 상승/하락 비율을 양수/음수로 전달받아 내부적으로 아이콘(상승/하락 기호)을 매핑해 화면에 동적으로 렌더링합니다.

2. **중단 현황 요약 (InfoCard 3종)**
   - "매출 현황", "회원 현황", "문의 현황"을 주제로 각 3~4개의 세부 항목 수치(예: 완료 금액, 취소 금액)를 묶어 리스트 형태로 제공합니다.
   - 하단에 위치한 바로가기 버튼(`onButtonClick`)을 클릭 시 `useNavigate`를 통해 연관된 실제 관리 페이지(예: `/orders`, `/app-members`)로 즉시 이동시킵니다.

3. **하단 차트 및 분석 (Chart Components)**
   - `OrderDetailsChart.tsx`: 주문 유형별, 결제수단별 등 다양한 기준의 주문 데이터를 탭 형식으로 스위칭하며 가로 프로그레스 바(막대 그래프) 형태로 시각화합니다.
   - `DailySalesChart.tsx`: 최근 7일간의 매출 추이를 세로 막대 그래프로 렌더링하여 트렌드를 제공합니다.

### 2-2. GA4 통계 현황 (`GA4Statistics.tsx`)
고객의 방문에서 결제에 이르는 여정과 애플리케이션 잔존율을 파악하기 위해 구성되었습니다.

1. **활동 사용자 분석 (DAU / MAU)**
   - 상단 4개의 `StatCard`를 통해 DAU(일간), WAU(주간), MAU(월간)를 렌더링하며, 잔존율을 뜻하는 `DAU/MAU 비율`을 강조 표기합니다.
2. **접속 환경 및 이탈률 요약 (InfoCard)**
   - 접속 기기별 비율(모바일/데스크톱)과 퍼널 첫 진입부터 결제까지의 주요 구간별 이탈률 %를 카드 형태로 요약합니다. 하단 버튼 클릭 시 각각 **디바이스 상세 분석**, **퍼널 상세 분석** 페이지로 이동합니다.
3. **고객 여정 퍼널 차트 (Funnel Visualization)**
   - 100% 진행도에서 점차 깎여 나가는 형태의 다단 컴포넌트입니다. `journeyData` 배열을 매핑하며 가장 첫 번째 파이프라인의 접속자 수(`maxUsers`)를 기준으로 다음 단계 전환 인원의 비율(`widthPerc`)을 인라인 스타일 퍼센테이지로 자동 연산해 게이지를 채웁니다.

### 2-3. GA4 디바이스 상세 분석 (`GA4DeviceDetail.tsx`)
고객들이 주로 어떠한 IT 환경에서 접속하는지를 다각도로 분석하기 위한 서브 뎁스(Depth) 페이지입니다.
- **다중 탭 인터페이스**: `디바이스`, `브라우저`, `OS`, `해상도` 4개의 탭을 제공하여, 각 분류 기준에 따라 정렬된 트래픽 점유율(데이터 테이블)과 분포도를 차트화하여 조회할 수 있도록 지원합니다. 모바일/PC UX 최적화 전략을 세우는 데 직접적으로 기여합니다.

### 2-4. GA4 퍼널 상세 분석 (`GA4FunnelDetail.tsx`)
진입부터 결제 완료까지 고객 여정의 병목 지점(Bottleneck)과 총 전환율을 파고들기 위한 서브 뎁스(Depth) 페이지입니다.
- **다중 탭 인터페이스**:
  1. **퍼널 요약**: 전체 진행 과정 플로우와 최종 전환/이탈률의 거시적 오버뷰 차트.
  2. **일별 추이**: 선택 기간 동안의 특정 퍼널 진입 및 전환율 추세를 날짜 라인으로 시각화.
  3. **이탈 분석**: 가장 많은 이탈이 발생하는 구간들의 원인과 트러블슈팅 데이터를 집중 타겟팅.
  4. **TOP 상품**: 장바구니에 가장 많이 담기거나 결제 전환에 크게 기여한 상위 랭킹 메뉴(상품) 리스트 제공.

## 3. 통신 및 에러 핸들링 (Data Fetching & Errors)

메인 대시보드는 `useDashboard()` 훅 내에서 React Query를 활용해 비동기로 데이터를 가져옵니다. 렌더링 및 통신 중 발생할 수 있는 주요 에러 시나리오와 내부 처리 방식, 그리고 사용자에게 안내되는 **상세 오류 메시지**는 다음과 같습니다.

### 3-1. 서버 요청 에러 (Network & Server)
서버 다운 및 네트워크 단절 시 표시되는 가장 우선순위가 높은 에러입니다.
- **오류 코드**: `ERR_NETWORK` / `50X SERVER_ERROR`
- **표시 메시지**: `대시보드 데이터를 불러오지 못했습니다. 네트워크 상태를 확인하거나 잠시 후 다시 시도해주세요.`
- **동작 방식**: 500 계열의 에러를 React Query가 캐치할 시 재시도(Retry)를 3회 수행한 후 최종 에러로 판명합니다. 스피너(`Spinner`) 렌더링을 중단하고 붉은색 스낵바(Toast) 메시지와 재시도 버튼을 띄웁니다.

### 3-2. 권한 부족 에러 (Forbidden)
비정상적인 URL 변조로 권한 밖의 서브 통계 페이지로 접근할 경우 응답합니다.
- **오류 코드**: `403 FORBIDDEN`
- **표시 메시지**: `해당 통계 자료를 열람할 권한이 없습니다. (접근 거부)`
- **동작 방식**: API 요청 전 라우터의 `<ProtectedRoute>` 레벨에서 1차로 커팅 후 `AccessDeniedPage`로 리다이렉트 처리합니다. 만약 백엔드에서 2차로 403을 반환하면 역시 루트 권한 검증 에러로 간주해 로그인 페이지로 튕겨냅니다.

### 3-3. 데이터 결측 및 포맷 에러 (Data Integrity)
서버에서 데이터를 성공적으로 내려받았지만 구성이 깨져 있어 프론트엔드 연산(+%, 통화 변환 등)에 실패할 때의 에러입니다.
- **오류 코드**: `DATA_PARSE_ERROR`
- **표시 메시지**: `집계된 통계 데이터가 없거나 형식이 올바르지 않아 차트를 그릴 수 없습니다.`
- **동작 방식**: 서버 통신에 성공하더라도 `data` 객체가 없거나 필수값(`dau`, `mau` 등)이 `null` 혹은 `undefined`인 경우 컴포넌트는 해당 에러 메시지가 표기되는 빈자리 템플릿 영역(Placeholder)을 대신 띄웁니다. 따라서 흰 창(White Screen of Death) 현상을 방지합니다.

### 3-4. 차트/퍼널 제로(Zero) 디비전 에러 방어
- 통계 처리에서 사용자 수가 0명인 경우 나눗셈 처리 렌더링 시 `NaN`이 도출되어 화면이 깨질 위험이 있습니다.
- 이를 막기 위해 컴포넌트 내부에서 `const maxUsers = journeyData[0]?.users || 1;` 와 같은 방어 코드를 적용해 무조건 에러를 우회(Bypass)하도록 설계되어 있습니다.

## 4. UI 및 컴포넌트 세부 정보

### 4-1. StatCard 컴포넌트 (`src/components/ui/StatCard.tsx`)
대시보드 및 GA4 화면 전반에 걸쳐 사용되는 최상위 공용 카드 컴포넌트입니다.
- **`format` 속성**: 데이터의 타입을 지정합니다. `currency` 지정 시 금액 알고리즘 함수가 가동되어 `100000000`의 숫자가 `1.0억원`으로 축약 렌더링됩니다.
- **`color` 속성**: 테마별 헥스 컬러맵 매핑(`primary`, `success`, `critical` 등)에 맞추어 카드의 우측 좌측 아이콘 데코레이션의 톤 앤 매너 색상을 적용합니다.
- **`change` 표기기능**: 내부적으로 `change === 0`일 때는 하이픈(`-`) 아이콘을, 양수일 때는 우상향 화살표를 노출하도록 조건부 렌더링 되어 있습니다.

### 4-2. 모바일 반응형 디자인 설계
두 페이지 모두 TailwindCSS의 반응형 유틸리티 클래스를 탑재하였습니다.
- 모니터 이상의 환경 (1024px ~): 헤더 요약 4열, 하단 분할 2열 등 와이드 스크린 구성.
- 태블릿 및 모바일 (~ 768px): 그리드 시스템 레이아웃이 단일 1열 트랙 스택 구조(`grid-cols-1`)로 수직 나열(Stacking)되며, 스크롤을 내려 정보를 순차적으로 탐색할 수 있도록 설계되어 있습니다.
