# 앱 사용자 통합 관리 (App Users)

## 📋 개요

앱 사용자의 전체 목록 조회, 세그먼트 데이터 추출, 사용자 그룹 관리를 포괄하는 통합 관리 모듈에 대한 명세서입니다.
사이드바 및 시스템 라우팅은 `/app-members` 기반으로 설계되어 있으나, 용어 혼선을 방지하고 명확한 지칭을 위해 기존 '회원(Member)' 관련 제반 문서를 본 **'사용자(User)'** 문서 하나로 통일 및 통합하여 규정합니다.

이 문서는 모든 사용자 관리 프론트엔드 파트 개발자 및 유지보수자가 참고할 수 있도록 다나까체로 작성되었습니다.

---

## 1. 하위 메뉴 및 라우팅 구조

| 라우트 | 탭/메뉴명 | 설명 |
| --- | --- | --- |
| `/app-members` | 전체 사용자 | 전체 앱 사용자 목록 및 상세 정보 열람 뷰 |
| `/app-members/inactive` | 비활동 사용자 | 3개월(90일) 이상 서비스 접속 이력이 없는 휴면 전환 대상 사용자 목록 |
| `/app-members/no-order` | 미주문 사용자 | 가입 후 현재까지 결제(주문) 이력이 단 한건도 없는 사용자 목록 |
| `/app-members/extract` | 사용자 데이터 추출 | 세그먼트 쿼리(필터식) 기반 목적별 대상 사용자 조회 및 CSV/Excel 추출 뷰 |
| `/app-members/groups` | 사용자 그룹 관리 | 추출된 타겟 데이터를 저장하거나, 매뉴얼하게 묶은 사용자 단위 논리 그룹 명단 뷰 |


---

## 2. 주요 기능 및 컴포넌트 동작

### 2-1. 사용자 기본 조회 및 상세 이력 (`AppMemberList`, `AppMemberDetail`)
- **목록 필터링 최적화**: 단일 `AppMemberList` 컴포넌트가 `filter` prop(`all` \| `inactive_90days` \| `no_order`)을 동적으로 전달받아 서로 다른 3개의 라우트에서 동일한 UI 테이블 뷰를 재사용하여 렌더링합니다.
- **다중 검색 체계**: 가입 상태(활성/비활성/탈퇴), 등급(VIP~Normal), 그리고 키워드(이름/ID/전화번호) 기반의 다차원 필터링을 지원합니다.
- **보안 마스킹 (Privacy)**: 열람 과정의 보안 원칙 준수를 위해 `MaskedData` 컴포넌트가 개입하여 휴대전화 문자 등 식별 가능한 민감 정보를 별 단위(`***`)로 마스킹 처리하여 리턴합니다.
- **7-Tab 상세 사이클 뷰어**: 사용자 리스트에서 특정 Row 클릭 시 라우팅 되는 상세 페이지 공간에서는 '기본정보', '이용내역', '주문내역', '포인트 이력', '쿠폰함', '이용권 구매', '푸시 알림' 총 7가지 종단 탭을 통해 파편화된 사용자 활동 사이클을 하나로 묶어 보여줍니다.

### 2-2. 마케팅 타겟 통계 및 사용자 데이터 추출 (`MemberExtract`)
- **멀티 뎁스 세그먼트 (Multi-depth Segment)**: 기본 등급, 주문 이력, 마케팅 액션 반응률 등 4단계의 탭을 통하여 정교한 타겟 사용자 세그먼트를 필터링합니다. (`useMemberSegment` 훅 사용)
- **컬럼 제어 및 반출**: 데이터 추출 완료 시 내보내고자 하는 필드(예: 이름, 연락처, 등급)만 스위칭하여 CSV 또는 Excel 포맷으로 추출합니다. 데이터 무단 덤프 방지를 위해 다운로드 훅(`useDirectExport`)은 감사 로깅(Audit Logging) 시스템과 연계됩니다.

### 2-3. 다이내믹 사용자 그룹 관리 (`MemberGroups`, `MemberGroupDetail`)
- **타겟 그룹핑 (Save state)**: 데이터 추출에서 완성된 고유 필터 세그먼트를 1회성 추출로 끝내지 않고, 곧바로 `사용자 그룹` 이라는 엔티티로 저장(`useCreateGroup`)해 관리 목록에 영구 편입시킵니다.
- **수동 멤버 할당**: 오프라인 이벤트 고객 등 필터식으로 산출할 수 없는 사용자의 경우 개별 사용자 상세 화면(`MemberGroupDetail`)에서 `AddMembersModal`을 띄워 검색/멀티 체크를 통해 리스트에 일괄 삽입/제거 할 수 있습니다.

## 3. 통신 및 데이터 아키텍처 (API)

사용자 정보 제어를 위해 연결해야 하는 필수 API 엔드포인트 맵핑입니다.

| 기능 분류 | 목적 | Method 및 엔드포인트 라우트 |
| --- | --- | --- |
| **목록 파싱** | 사용자 리스트 조회 및 Limit 페이징 연산 | `GET /api/app-members?filter={type}` |
| **현황 집계** | 탭 표시용 활성/휴면 사용자 통계 렌더링 | `GET /api/app-members/stats` |
| **단일 추적** | 특정 ID 사용자 상세/주문 내역 등 전체 열람 | `GET /api/app-members/:id` |
| **타겟 추출** | 조건부 세그먼트 분석 및 타겟팅 JSON 로드 | `POST /api/app-members/segment` |
| **시트 반출** | 다운로드 권한 검증 및 파일 바이너리 리턴 | `POST /api/app-members/export` |
| **그룹 CRUD** | 사용자 논리 그룹 형성 및 구성원 삽입/제거 | `POST /api/member-groups/:id/members` 등 |

## 4. UI 렌더링 플로우 및 상호작용 (요약)

1. **LNB 로드 및 페이징**: 사이드바 하위 메뉴 진입 시, 상단 요약 배지(전체 회원 수 등 `useAppMemberStats` 호출)가 그려짐과 동시에 리스트(`useAppMembers`) 렌더링을 시작합니다.
2. **이벤트 드리븐 (타겟 도출)**: `사용자 데이터 추출` 메뉴에서 체크박스로 필터식을 만들고 `조회` 액션을 발생시키면, 실시간으로 하단 예상 추출 사용자 수가 바인딩됩니다.
3. **그룹 스토리지 보관**: 예상 인원을 확인 후 `그룹으로 저장`을 실행하면 `GroupFormModal`이 마운트 되며, 명칭 입력 후 승인 시 즉각 데이터베이스 그룹 테이블에 반영되어 `사용자 그룹 관리` 화면에 노출됩니다.
4. **마케팅 액션 릴레이**: 형성된 그룹 정보는 차후 `마케팅관리 > 앱푸시관리` 혹은 `쿠폰관리` 메뉴의 대상자 선택 드롭다운 콤보박스 트리거 소스로 다시 페칭되어 선순환 구도를 이룹니다.
