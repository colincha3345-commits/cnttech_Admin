# 쿠폰 관리

## 개요

쿠폰은 고객의 결제 금액에서 일정 금액 또는 비율을 차감해주는 할인 수단이다.
쿠폰 관리 페이지에서는 쿠폰 자체의 "할인 정의"만 다루며, 어떤 조건에서 어떤 고객에게 쿠폰을 발행할지는 캠페인에서 설정한다.

즉, 쿠폰은 **"무엇을 얼마나 할인해줄 것인가"** 만 정의하고, **"누구에게 언제 어떻게 지급할 것인가"** 는 캠페인의 영역이다.

---

## 쿠폰의 종류

쿠폰은 `applyScope` 필드에 따라 세 종류로 나뉜다. 이 값에 따라 할인이 적용되는 대상이 달라진다.

### 1. 장바구니 할인 쿠폰 (`cart_total`)

장바구니에 담긴 전체 상품의 합산 금액에 대해 할인을 적용하는 쿠폰이다.

- 고객이 장바구니에 여러 상품을 담고 결제할 때, 상품 합계 금액에서 할인이 차감된다.
- 가장 일반적인 쿠폰 형태이다.
- 예: "장바구니 금액에서 3,000원 할인", "장바구니 합계의 10% 할인"

### 2. 특정 상품 할인 쿠폰 (`specific_product`)

관리자가 지정한 특정 상품에만 할인이 적용되는 쿠폰이다.

- 쿠폰 생성 시 `applicableProductIds`에 대상 상품 ID 목록을 지정해야 한다.
- 고객의 장바구니에 해당 상품이 포함되어 있을 때만 쿠폰을 사용할 수 있다.
- 하나의 상품에는 하나의 활성 쿠폰만 지정할 수 있다. 이미 다른 쿠폰이 적용된 상품은 선택할 수 없다(ExcludedProduct).
- 예: "후라이드치킨 2,000원 할인", "신메뉴 출시 기념 15% 할인"

### 3. 배달비 할인 쿠폰 (`delivery_fee`)

배달 주문 시 발생하는 배달비에 대해 할인을 적용하는 쿠폰이다.

- 배달 주문(`delivery`)에서만 의미가 있다. 포장(`pickup`) 주문에는 배달비가 없으므로 적용되지 않는다.
- 할인 금액이 실제 배달비보다 큰 경우, 배달비를 0원까지만 차감한다(음수 불가).
- 예: "배달비 무료 쿠폰", "배달비 1,000원 할인"

---

## 할인 방식

쿠폰의 할인 금액을 계산하는 방식은 두 가지이다.

### 정액 할인 (`fixed`)

고정된 금액을 차감하는 방식이다.

- `discountValue`에 원(₩) 단위의 금액을 입력한다.
- 할인 금액은 항상 동일하다. 주문 금액이 크든 작든 같은 금액을 할인한다.
- `maxDiscountAmount`는 사용하지 않는다(null).
- 예: `discountValue: 3000` → 항상 3,000원 할인

### 정률 할인 (`percentage`)

적용 대상 금액의 일정 비율을 차감하는 방식이다.

- `discountValue`에 퍼센트(%) 값을 입력한다. 1~100 사이의 정수이다.
- 할인 금액은 적용 대상 금액에 비례하므로 주문 금액이 클수록 할인도 커진다.
- 과도한 할인을 방지하기 위해 `maxDiscountAmount`로 최대 할인 금액을 설정할 수 있다.
- 예: `discountValue: 20, maxDiscountAmount: 10000` → 20% 할인하되, 최대 10,000원까지만 할인

### 할인 금액 계산 예시

| 상황 | 할인 방식 | 설정값 | 주문금액 | 실제 할인 |
| --- | --- | --- | --- | --- |
| 정액 | `fixed` | 3,000원 | 15,000원 | 3,000원 |
| 정액 | `fixed` | 3,000원 | 50,000원 | 3,000원 |
| 정률 | `percentage` | 20%, 최대 10,000원 | 30,000원 | 6,000원 (30,000 × 20%) |
| 정률 | `percentage` | 20%, 최대 10,000원 | 80,000원 | 10,000원 (상한 적용) |

## 최소 주문 금액

`minOrderAmount` 필드는 쿠폰을 사용하기 위한 최소 주문 금액 조건이다.

- 고객의 주문 금액(할인 전)이 이 값 이상이어야 쿠폰을 적용할 수 있다.
- 0으로 설정하면 금액 제한 없이 모든 주문에 사용 가능하다.
- 예: `minOrderAmount: 20000` → 주문 금액이 20,000원 이상일 때만 쿠폰 사용 가능

이 조건은 너무 적은 금액의 주문에 과도한 할인이 적용되는 것을 방지하기 위한 장치이다.

---

## 주문 유형 제한

`orderType` 필드는 쿠폰을 사용할 수 있는 주문 유형을 제한한다.

| 값 | 설명 |
| --- | --- |
| `all` | 배달, 포장 모든 주문에서 사용 가능 |
| `delivery` | 배달 주문에서만 사용 가능. 포장 주문 시 쿠폰이 표시되지 않는다 |
| `pickup` | 포장 주문에서만 사용 가능. 배달 주문 시 쿠폰이 표시되지 않는다 |


배달비 할인 쿠폰(`delivery_fee`)은 배달 주문에서만 의미가 있으므로, 일반적으로 `orderType`을 `delivery`로 설정한다.

## 유효 기간

쿠폰을 사용할 수 있는 기간을 설정한다.

- `startDate`: 쿠폰 사용 가능 시작일. 이 날짜부터 고객이 쿠폰을 사용할 수 있다.
- `endDate`: 쿠폰 사용 가능 종료일. 이 날짜가 지나면 쿠폰은 자동으로 만료(`expired`) 상태가 된다.
- `autoDelete`: `true`로 설정하면 만료된 쿠폰을 고객의 쿠폰함에서 자동으로 제거한다. `false`이면 만료 상태로 남겨둔다.

유효 기간은 쿠폰 자체의 "사용 가능 기간"이다. 고객에게 쿠폰을 언제 지급할지는 캠페인에서 결정한다.

---

## 사용 제한

쿠폰의 수량과 사용 횟수를 제한하는 설정이다.

### 총 발행 수량 (`totalCount`)

- 이 쿠폰을 최대 몇 장까지 발행할 수 있는지 지정한다.
- `null`로 설정하면 수량 제한 없이 무제한 발행할 수 있다.
- `usedCount`가 `totalCount`에 도달하면 쿠폰은 `exhausted`(소진됨) 상태가 된다.
- 예: `totalCount: 1000` → 최대 1,000장까지 발행, 1,000장 모두 사용되면 자동 소진

### 1인 1회 사용 제한 (`singleUsePerMember`)

- `true`로 설정하면 한 명의 회원이 이 쿠폰을 딱 한 번만 사용할 수 있다.
- `false`로 설정하면 같은 회원이 여러 번 사용할 수 있다(수량이 남아 있는 한).
- 대부분의 쿠폰은 `true`로 설정하여 1인 1회 제한을 건다.

## 사용 가능 스케줄

쿠폰을 사용할 수 있는 요일과 시간대를 제한하는 설정이다.

### 사용 가능 요일 (`availableDays`)

- 0(일요일)부터 6(토요일)까지의 숫자 배열이다.
- `[0, 1, 2, 3, 4, 5, 6]`이면 매일 사용 가능 (기본값).
- `[1, 2, 3, 4, 5]`이면 평일에만 사용 가능.
- `[0, 6]`이면 주말에만 사용 가능.

### 사용 가능 시간대 (`availableTimeRanges`)

- 쿠폰을 사용할 수 있는 시간 범위를 지정한다.
- `startTime`과 `endTime`은 `HH:mm` 형식이다.
- 기본값은 `00:00 ~ 23:59`(종일 사용 가능)이다.
- 예: `11:00 ~ 14:00`으로 설정하면 점심시간에만 쿠폰 사용 가능.

이 설정은 특정 시간대의 매출을 촉진하기 위한 목적으로 사용된다. 예를 들어 "평일 점심 할인 쿠폰"이나 "주말 저녁 할인 쿠폰" 등을 만들 수 있다.

## 가맹점 제한

쿠폰을 사용할 수 있는 가맹점을 제한하는 설정이다.

| `storeRestriction.type` | 동작 |
| --- | --- |
| `all` | 모든 가맹점에서 사용 가능 (기본값) |
| `include` | `storeIds`에 포함된 가맹점에서만 사용 가능. 목록에 없는 가맹점에서는 쿠폰이 표시되지 않는다 |
| `exclude` | `storeIds`에 포함된 가맹점에서는 사용 불가. 나머지 가맹점에서는 사용 가능 |


예시:
- 전체 가맹점 대상 프로모션 → `type: 'all'`
- 강남점, 홍대점에서만 사용 가능한 지역 한정 쿠폰 → `type: 'include', storeIds: ['store-1', 'store-2']`
- 특정 가맹점이 프로모션에서 제외 요청한 경우 → `type: 'exclude', storeIds: ['store-3']`

## 정산 비율

쿠폰으로 인한 할인 금액을 본사와 가맹점이 어떤 비율로 부담할지 설정한다.

- `headquartersRatio`: 본사가 부담하는 비율 (%)
- `franchiseRatio`: 가맹점이 부담하는 비율 (%)
- 두 값의 합은 항상 100이어야 한다.

예시:
- `headquartersRatio: 100, franchiseRatio: 0` → 본사가 할인 금액 전액 부담 (기본값)
- `headquartersRatio: 80, franchiseRatio: 20` → 본사 80%, 가맹점 20% 부담
- `headquartersRatio: 50, franchiseRatio: 50` → 본사와 가맹점이 반반 부담

3,000원 할인 쿠폰에서 본사 80% / 가맹점 20% 설정이면, 본사가 2,400원, 가맹점이 600원을 부담한다.

## 쿠폰 상태

쿠폰은 다음 네 가지 상태 중 하나를 가진다.

| 상태 | 코드 | 설명 |
| --- | --- | --- |
| 활성 | `active` | 정상적으로 사용 가능한 상태. 고객에게 노출되고 결제 시 적용할 수 있다 |
| 비활성 | `inactive` | 관리자가 수동으로 비활성화한 상태. 고객에게 노출되지 않는다. 다시 활성화할 수 있다 |
| 만료됨 | `expired` | `endDate`가 지나서 자동으로 만료된 상태. 더 이상 사용할 수 없다 |
| 소진됨 | `exhausted` | `usedCount`가 `totalCount`에 도달하여 수량이 모두 소진된 상태 |


---

## 쿠폰과 캠페인의 관계

쿠폰 관리 페이지에서 생성한 쿠폰은 그 자체로 "할인 조건의 정의"일 뿐이며, 실제로 고객에게 쿠폰이 발행되려면 캠페인을 통해야 한다.

- **쿠폰이 정의하는 것**: 할인 종류, 할인 금액/비율, 적용 대상, 사용 조건, 유효 기간
- **캠페인이 정의하는 것**: 어떤 고객에게(타겟팅), 어떤 조건에서(트리거), 언제(스케줄), 이 쿠폰을 지급할 것인지

예를 들어 "첫 주문 3,000원 할인 쿠폰"을 만든다면:
1. 쿠폰 관리에서 "정액 3,000원, 최소주문 15,000원, 장바구니 할인" 쿠폰을 생성한다.
2. 캠페인에서 "첫 주문 완료 시 해당 쿠폰을 자동 지급"하는 룰을 설정한다.

이렇게 분리함으로써 하나의 쿠폰을 여러 캠페인에서 재사용하거나, 캠페인 조건만 변경하여 쿠폰의 발행 전략을 유연하게 운영할 수 있다.

## 유효성 검증 규칙

쿠폰 등록/수정 시 다음 항목을 검증한다.

| 검증 항목 | 조건 | 에러 메시지 |
| --- | --- | --- |
| 쿠폰명 | 빈 문자열이면 안 됨 | "쿠폰명을 입력해주세요." |
| 할인 금액/비율 | 0 이하이면 안 됨 | "할인 금액/비율을 입력해주세요." |
| 정률 상한 | 정률 할인일 때 100을 초과하면 안 됨 | "할인율은 100%를 초과할 수 없습니다." |

## 실제 쿠폰 설계 예시

### 예시 1: 첫 주문 환영 쿠폰

> "첫 주문 시 20% 할인, 최대 10,000원까지"

```
이름: 첫 주문 쿠폰
종류: 장바구니 할인 (cart_total)
할인 방식: 정률 20%
최대 할인: 10,000원
최소 주문 금액: 20,000원
주문 유형: 전체 (배달/포장 모두)
유효 기간: 2024-01-01 ~ 2024-12-31
총 발행 수량: 1,000장
1인 1회 제한: 예
스케줄: 매일 종일
가맹점: 전체
정산: 본사 100%
```

이 쿠폰은 20,000원 이상 주문하는 고객에게 20% 할인을 제공하되, 최대 10,000원까지만 할인한다.
35,000원 주문 시 7,000원 할인, 60,000원 주문 시 10,000원 할인(상한 적용).
캠페인에서 "회원가입 후 첫 주문 완료 시 자동 지급" 조건을 설정한다.

### 예시 2: 배달비 무료 쿠폰

> "배달 주문 시 배달비 전액 무료"

```
이름: 배달비 무료 쿠폰
종류: 배달비 할인 (delivery_fee)
할인 방식: 정액 5,000원
최소 주문 금액: 15,000원
주문 유형: 배달 전용
유효 기간: 2024-03-01 ~ 2024-05-31
총 발행 수량: 무제한
1인 1회 제한: 아니오 (반복 사용 가능)
스케줄: 평일만 (월~금)
가맹점: 전체
정산: 본사 50%, 가맹점 50%
```

배달비가 3,000원인 경우 3,000원 할인(배달비 0원), 배달비가 5,000원인 경우 5,000원 할인(배달비 0원).
할인 금액이 배달비보다 크면 배달비를 0원까지만 차감하고, 초과분은 적용하지 않는다.
정산은 할인된 배달비를 본사와 가맹점이 반반 부담한다.

### 예시 3: 특정 메뉴 프로모션 쿠폰

> "신메뉴 후라이드치킨 2,000원 할인"

```
이름: 신메뉴 출시 기념 쿠폰
종류: 특정 상품 할인 (specific_product)
할인 방식: 정액 2,000원
최소 주문 금액: 0원
주문 유형: 전체
적용 상품: [후라이드치킨(product-101)]
유효 기간: 2024-04-01 ~ 2024-04-30
총 발행 수량: 500장
1인 1회 제한: 예
스케줄: 매일 종일
가맹점: 강남점, 홍대점만 (include)
정산: 본사 80%, 가맹점 20%
```

고객의 장바구니에 "후라이드치킨"이 포함되어 있을 때만 쿠폰을 사용할 수 있다.
다른 상품만 주문하는 경우 이 쿠폰은 적용 불가하다.
강남점과 홍대점에서만 사용할 수 있는 지역 한정 프로모션이다.

### 예시 4: 주말 저녁 타임 할인 쿠폰

> "주말 저녁 시간대 10% 할인"

```
이름: 주말 저녁 할인 쿠폰
종류: 장바구니 할인 (cart_total)
할인 방식: 정률 10%
```
