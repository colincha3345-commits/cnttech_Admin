# 매장(Store) 관리 시스템 기획 명세서

본 문서는 관리자 대시보드 내 **매장(Store) 관리 카테고리**의 프론트엔드 및 백엔드 개발 시 필요한 세부 필수 항목과 페이지 프로세스를 정의한 문서입니다. 매장은 영업 상태와 배달 설정의 실시간 연동이 매우 중요한 도메인입니다.

---

## 1. 페이지 프로세스 (Page Process)

1. **매장 목록 조회 (List View)**
   - 관리자가 매장 관리 메뉴에 진입하면 현재 운영중(`active`)인 매장을 우선 노출합니다.
   - 검색 바에서 지역, 계약 상태, 매장명, 점주명 등의 조합 필터 검색을 제공합니다.
   - 상태 토글 스위치 비노출 (상세에서만 변경 권장). 리스트에서는 각 매장의 상태, 영업시간, 배달/포장 여부 배지를 간략히 노출합니다.

2. **매장 등록 및 상세 관리 (Create & Detail View)**
   - 신규 매장 등록 클릭 시 탭 형태의 복합 입력 폼(기본정보, 영업/배달설정, 연동, 노출)을 순차적으로 기입하도록 유도합니다.
   - 각 탭 간 이동 시 브라우저 내 상태(State) 유지를 지원하고, [저장] 버튼 클릭 시 전체 데이터 폼 검증 후 API를 호출합니다.
   - POS, PG 연동 정보의 경우 별도 [연동 테스트] 버튼을 두어 정상 작동 여부를 사전에 스크리닝 하도록 합니다.

3. **실시간 영업 제어 (Operating Control)**
   - 상세 페이지 상단의 '앱 운영 상태' (준비중, 휴식, 영업중) 등은 매장과 실시간 동기화되어야 하며, 관리자가 직접 강제 토글 제어가 가능해야 합니다.

## 2. 세부 개발 명세

### 2.1. 프론트엔드 (Frontend) 개발 요건

| 기능 / 필드명 | 입력/노출 형태 | 필수 여부 | 글자수 / 제약조건 | 비고 (UI/UX) |
| :--- | :--- | :---: | :--- | :--- |
| **매장명 (name)** | Input | Y | 최대 50자 | 텍스트 입력창. 상단 제목과 연동되어 실시간 변경 표출합니다. |
| **사업자번호** | Input | Y | 10자리 고정 | 000-00-00000 포맷에 맞게 입력 시 하이픈 자동 생성 처리합니다. |
| **계좌 정보** | Select + Input | Y | 은행별 계좌길이 다름 | 입금 은행 Select Box와 계좌번호 Input을 병렬 배치합니다. |
| **계약 일자** | DatePicker | Y | YYYY-MM-DD | 시작일과 종료일을 묶어서 Range Picker 형태로 UI를 제공합니다. |
| **영업 시간** | TimePicker | Y | HH:mm | 요일별 `openTime`과 `closeTime`을 입력받으며, 마감시간이 자정 이후일 경우 알림 문구 표출합니다. |
| **배달비 설정** | Dynamic List | Y | 숫자(원) 제한 | 거리별/금액별 배달비를 동적으로 추가/삭제 가능한 '+ 행 추가' UI로 구현합니다. |
| **예약 기능 설정** | Switch + Input | N | 숫자(시간) 제한 | 예약기능 ON 토글 시 '현재 시간 + N 분부터'를 입력받는 인풋이 Fade-in 표출되도록 합니다. |

**[UI/UX 상호작용 제약사항]**
- 사업자번호는 OnBlur 시 정규식에 따른 1차 유효성 검사 및 '이미 등록된 번호' 방어 로직(API 확인)을 띄웁니다.
- 위치/지점 입력 카카오맵 또는 네이버 지도 API 주소 연동으로 위도(Latitude)/경도(Longitude) 자동 바인딩 처리합니다.

### 2.2. 백엔드 (Backend) 개발 요건

| 데이터베이스 필드 | 데이터 타입 | 필수 여부 | 글자수 / 제약조건 | 비고 (API 설계) |
| :--- | :--- | :---: | :--- | :--- |
| **storeId (PK)** | UUID | Y | 36자 | 가맹점 식별 고유키로 사용합니다. |
| **status** | Enum | Y | - | 'active', 'inactive', 'pending', 'terminated' 만 허용합니다. |
| **businessNumber** | String | Y | 10자 (숫자) | 중복값 검증이 반드시 필요합니다 (Unique Constraint). |
| **address JSON** | JSON | Y | - | zipCode, address, lat, lng 객체를 통채로 저장 및 반환 권장합니다. |
| **operatingInfo JSON**| JSON | Y | - | 평일/휴일 영업시간, 배달/포장 설정 내역 등 방대한 영업데이터를 구조화하여 담습니다. |
| **deliveryFee** | Integer | Y | 0 이상 | 기본 배달비는 필수이며, 음수 입력 시 400 Bad Request 리턴합니다. |

**[API 및 비즈니스 로직 제약사항]**
- **상태 변경/임시 휴점 API (`PATCH /api/stores/{id}/status`)**
  - 상태 변경, 특히 영업중 -> 임시휴업(`temporarily_closed`) 등으로의 전환은 점주 POS앱 연동을 고려하여 매우 빠르고 가벼운 전용 단일 API 분리 운영을 권장합니다.
- **예약 가능 시간 유효성 (Backend Validation)**
  - 요청된 `reservationLeadTimeMinutes` 값이 매장의 실제 마감시간(`closeTime`)을 초과하게 세팅되는 모순을 서버의 저장 시점(Save)에 반드시 밸리데이션(Validation)하여 예외를 반환해야 합니다.
- **연락처 계정 연동 로직 (StaffLink)**
  - 가맹점 생성 후(Post) 해당 가맹점을 소유하는 점주(Owner) 어드민 계정 매핑 및 권한(Role) 인서트 작업이 트랜잭션 단위로 함께 완료되어야 합니다.
