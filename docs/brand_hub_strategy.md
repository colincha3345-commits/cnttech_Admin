# 브랜드 허브(Brand Hub) 전략 기획서 및 상위 시스템 관계 정의

**브랜드오더 관리자 시스템(Admin Dashboard) 기획서**
작성일: 2026년 3월 5일 (최근 수정일: 2026년 3월 17일)

---

## 1. 개요: 브랜드 허브란 무엇인가?

우리가 현재 만들고 있는 **관리자 대시보드(Admin Dashboard)** 시스템은 대규모 SaaS 플랫폼 아키텍처에서 정중앙에 위치하는 **'브랜드 허브(Brand Hub)'** 역할을 합니다. 

간단히 비유하자면, 거대한 백화점(가상의 SaaS 플랫폼) 전체를 관리하는 사람이 **'슈퍼어드민(플랫폼 센터)'**이라면, 백화점 안에 입점한 개별 명품 매장이나 프랜차이즈 브랜드 매장의 점장/본사 책임자가 바로 우리 시스템을 사용하는 **'브랜드 관리자'**입니다.

브랜드 허브는 자신의 브랜드 정체성을 유지하고, 밑에 딸린 수십~수백 개의 개별 가맹점(운영 접점)들을 효과적으로 통제하면서 비즈니스를 주도적으로 이끌어나가는 **'본사 전용 지휘 통제소'**입니다.

---

## 2. 슈퍼어드민(부모)과의 관계 및 상속 정책

브랜드 허브는 절대적으로 독립적인 시스템이 아니라, 더 거대한 생태계를 관리하는 **슈퍼어드민의 정책 테두리 안에서 움직이면서도 최대한의 자율성을 부여받는 관계**를 가집니다. 특히 중요한 핵심 데이터는 슈퍼어드민으로부터 상속받아 사용합니다.

### 🏢 브랜드 입점 등록 시 핵심 상속 정보 (최초 셋팅)
- 브랜드 허브가 활성화되기 위해서는 슈퍼어드민에서 **'브랜드 입점 등록'**이 완료되어야 합니다.
- **사업자 연계:** 하나의 사업자는 다수의 브랜드를 운영할 수 있습니다. 슈퍼어드민에서 사업자등록증 및 통장사본을 업로드하여 등록된(증명된) 사업자와 계열 브랜드를 맵핑하여 브랜드 허브의 기초 정산/법적 기준이 상속됩니다.
- **운영/연동 제원 세팅:** 브랜드 허브를 최초 사용할 **관리자 접속 계정**, 공식 **입점일(계약 시작일)** 등과 함께 **결제모듈(PG)사의 브랜드코드, 사용 포스(POS)사의 브랜드코드, e쿠폰(금액권/교환권)사의 관리 코드**가 입력되어 상속됩니다. 이를 통해 각 서드파티 서비스들과 연동할 수 있는 식별값을 갖춥니다.

### 🛡️ 상호 보완적인 거버넌스 (통제와 자율의 조화)
- **슈퍼어드민(부모):** 전체 플랫폼의 안정성과 서버 자원을 관리합니다. "A 등급 요금제를 쓰는 브랜드는 푸시 알림 기능을 최대 10만 건까지만 쓸 수 있다" 같은 전역(Global) 정책과 울타리를 설정합니다.
- **브랜드 허브(자식):** 슈퍼어드민이 제공한 울타리 안에서, 우리 브랜드만의 독자적인 쿠폰을 발급하고, 가맹점별로 배달/포장 설정을 조율하며, 자체 VIP 회원 등급을 자유롭게 만들어 운영합니다.

### 🔗 가맹점 계약정보 상속 (부모에게 상속받는 케이스)
- **가맹점 계약정보(기본 제원):** 브랜드에 속한 각 가맹점의 핵심 계약 정보인 **포스(POS) 연동 정보, PG(결제대행사) 연동 정보, 사업자 등록 정보 등 상위 계약 데이터는 전적으로 슈퍼어드민(부모) 시스템에서 등록되어 브랜드 허브로 상속**됩니다. 브랜드 관리자는 이 상속된 기준 정보를 바탕으로 메뉴 및 운영 정책을 매핑하고 가맹점을 관리하게 됩니다.

### 📈 요금제에 따른 유연한 상호작용 (Business Keystoning)
- 브랜드 허브는 요금제(Standard, Professional 등)에 따라 화면에 보이는 메뉴나 기능이 달라집니다. 플랫폼 중심의 과금 체계와 연동되어 비즈니스 규모에 맞춘 확장을 지원합니다.

## 3. 브랜드 허브의 핵심 기획 의도와 역할, 반영 시점

### ① 메뉴 및 매장정보의 "표준화" 및 하위 "통제"
- **역할:** 본사(브랜드 허브)에서 메뉴, 옵션, 프로모션(할인/쿠폰) 정책을 일괄 표준화하여 하위 가맹점으로 내려보냅니다.
- **업데이트 프론트 반영 시점:** 관리자가 메뉴 정보, 가격 변경, 매장 정보 변경 등을 등록/수정/삭제하는 경우, 프론트에 실시간 웹소켓으로 부하를 주는 대신 **"새로고침 및 페이지 전환 시"** 최신 데이터를 반영하는 것을 표준 업데이트 시점으로 명기합니다.
- **예외적 허용:** 단, 지역 상권의 특수성을 고려해 가맹점 스스로가 제어할 수 있는 부분(Override 기능)을 일부 열어주어 유연함을 보장합니다.

### ② 마케팅과 캠페인의 "중앙 집중화"
- **역할:** 전체 통합 회원을 대상으로 조건(가입 시, N번째 주문 시, 생일 등)에 맞춘 **자동화된 혜택(Benefit Campaign)**을 중앙에서 세팅하여 하달합니다.

### ③ 상권관리 통합 및 소상권 제어
- **역할:** 프랜차이즈 권역 관리를 위해 대분류 상권(시/도, 구 단위 등)부터 세부 소상권까지 계층적으로 관리할 수 있도록 지원합니다.
- **소상권 그리기 유형:** 관리자가 지도상에서 각 가맹점의 실질적인 배달/영업 권역인 '소상권'을 지정할 때, 더욱 직관적이고 정확한 설정이 가능하도록 **다각형(Polygon), 원형(Circle), 반경(Radius)** 등의 그리기 유형을 명시적 툴로 제공합니다.

### ④ 데이터 통합 가시성 확보 (대시보드 인사이트)
- **역할:** 각기 다른 매장(POS)과 고객 앱에서 올라오는 트랜잭션(주문, 매출) 데이터를 실시간으로 모아, 본사 관리자가 보기 쉬운 단위로 변환해 보여줍니다.

## 4. 전체 카테고리의 사용자별 유즈케이스 (권한별 사용 케이스)

브랜드 허브를 구성하는 전체 기능을 시스템에 접속하는 3단계 레벨의 사용자 권한별로 정의합니다.

### [권한 체계 정의]
1. **최고 관리자 (Brand Owner):** 브랜드 본사의 총괄 책임자
2. **운영 관리자 (Operation Manager):** 영업/마케팅/메뉴 담당 본사 실무자
3. **가맹점주 (Store Owner):** 개별 매장을 운영하는 점주

### 카테고리별 유즈케이스
| 시스템 카테고리 | 최고 관리자 (Brand Owner) | 운영 관리자 (Operation Manager) | 가맹점주 (Store Owner) |
| :--- | :--- | :--- | :--- |
| **대시보드 / 통계** | 전 가맹점 통합 매출/동향 열람, 전사 지표 확인 | 담당 권역/브랜드 매출 및 마케팅 성과 통계 조회 | 자기 매장의 매출, 주문 현황, 고객 리뷰 통계만 조회 |
| **계약 / 매장 관리** | (슈퍼어드민 상속) PG/POS 계약정보 조회, 신규 매장 승인 | 매장별 영업시간, 상태(오픈/마감) 관리 리딩 | 본인 매장 영업시간 임시 변경, 주문 접수 상태 On/Off |
| **상권 관리 (소상권)** | 전국 상권 맵핑 마스터 권한, 소상권 정책 승인 | 소상권 분할/통합, 지도에서 다각형/원형 그리기 유형으로 권역 셋팅 | 자기 매장에 할당된 배달 권역(소상권) 조회 및 면적 확인 |
| **메뉴 / 상품 관리** | 브랜드 통합 메뉴 정책, 카테고리 체계 확립 | 신메뉴 등록, 가격 수정 (새로고침/페이지 전환 반영), 옵션 그룹 설정 | (본사 정책 범위 내) 품절 처리, 일부 메뉴 노출/비노출 제어 |
| **마케팅 / 캠페인** | 통합 프로모션 예산 승인, 전사 쿠폰 발행 | 타겟 고객별 자동화 쿠폰 세팅, 이벤트 페이지 관리 | (본사 허용 시) 자기 매장 전용 게릴라 할인 적용 |

## 5. 백엔드 관련 코드 예시 (데이터 상속 및 반영 메커니즘)

정책 문서에 명시된 정책(상속/반영 시점)을 시스템으로 구현할 때 참고할 수 있는 백엔드 API 슈도/예시 코드입니다.

```typescript
// [부모에게 상속받는 케이스] 슈퍼어드민 상속 계약정보(POS, PG 등) 조회 예시
@Get('/api/v1/brand/stores/:storeId/contract-info')
async getStoreContractInfo(@Param('storeId') storeId: string) {
  // Brand 자체 DB가 아닌, Global DB 또는 SuperAdmin 프록시를 통해 상속된 기본 제원을 호출합니다.
  const parentInheritedData = await this.superAdminService.getInheritedContractData(storeId);
  
  return {
    storeId: parentInheritedData.storeId,
    posConfig: parentInheritedData.posConfig,         // 슈퍼어드민 상속 POS 정보
    pgProvider: parentInheritedData.pgProvider,       // 슈퍼어드민 상속 PG 연동 정보
    businessNumber: parentInheritedData.businessNumber,
    isInherited: true
  };
}

// [업데이트 반영 시점 케이스] 메뉴 및 매장정보 업데이트 예시 (페이지 전환 시 반영)
@Put('/api/v1/brand/menu/:menuId')
async updateMenuInfo(@Param('menuId') menuId: string, @Body() updateDto: MenuUpdateDto) {
  // 메뉴 정보를 업데이트
  const updatedMenu = await this.menuService.update(menuId, updateDto);
  
  // 프론트엔드가 페이지 전환 또는 새로고침 시 변경을 인지하고 패치하도록 전역 버전 캐시 갱신
  await this.cacheService.incrementGlobalVersion('MENU_VERSION');

  return {
    success: true,
    data: updatedMenu,
    message: "메뉴 정보가 업데이트되었습니다. 클라이언트는 새로고침 및 페이지 전환 시 변경사항을 최신화합니다."
  };
}
```

## 6. 요약 (Conclusion)

결론적으로 지금 우리가 구성하고 있는 **브랜드 허브(본사 관리자 시스템)**는 **"위로는 슈퍼어드민의 보호와 계약 정보(포스, PG 등)를 견고하게 상속받으면서, 아래로는 소상권 그리기와 실각 최신화(새로고침 시) 정책을 통해 수많은 가맹점들을 자율적이면서도 일관성 있게 묶어 통제하는 최전선 본부"**입니다. 또한 다양한 권한의 사용자들이 각자의 유즈케이스에 맞게 시스템을 활용하여 플랫폼 위에 독자적인 생태계를 효과적으로 경영할 수 있도록 돕습니다.
