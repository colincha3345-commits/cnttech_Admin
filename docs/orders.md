# 주문관리

## 개요

주문관리는 고객이 앱, 웹, 키오스크, POS 등 다양한 채널에서 접수한 주문을 조회하고 관리하는 기능이다. 관리자는 주문 목록을 필터링하여 검색하고, 상세 정보를 확인하며, 주문 상태를 변경하거나, 필요한 경우 주문을 취소할 수 있다. 주문에 메모를 남기는 기능과 조회 결과를 엑셀로 내보내는 기능도 제공한다.

---

## 주문 유형

주문은 `orderType` 필드에 따라 세 가지로 구분된다.

| 유형 | 코드 | 설명 |
| --- | --- | --- |
| 배달 | `delivery` | 고객이 지정한 주소로 배달하는 주문. 배달비가 부과된다 |
| 포장 | `pickup` | 고객이 매장에 방문하여 포장해 가는 주문. 배달비가 없다 |
| 매장식사 | `dine_in` | 고객이 매장 내에서 식사하는 주문. 배달비가 없다 |

## 주문 채널

주문이 접수된 경로를 나타낸다.

| 채널 | 코드 | 설명 |
| --- | --- | --- |
| 앱 | `app` | 모바일 앱을 통한 주문 |
| 키오스크 | `kiosk` | 매장 내 키오스크를 통한 주문 |
| POS | `pos` | 매장 POS 시스템을 통한 주문 |
| 웹 | `web` | 웹사이트를 통한 주문 |

## 주문 상태

주문은 접수부터 완료까지 다음 상태를 순서대로 거친다. 취소는 어느 단계에서든 발생할 수 있지만, 관리자 화면에서의 취소 처리는 `pending`과 `confirmed` 상태에서만 가능하다.

| 상태 | 코드 | 설명 | 다음 상태 |
| --- | --- | --- | --- |
| 결제 대기 | `pending` | 주문이 접수되었으나 아직 확인되지 않은 상태 | confirmed |
| 주문 확인 | `confirmed` | 매장에서 주문을 확인한 상태 | preparing |
| 준비 중 | `preparing` | 메뉴를 조리하고 있는 상태 | ready |
| 준비 완료 | `ready` | 조리가 완료되어 배달/픽업 대기 중인 상태 | delivered |
| 배달 중 | `delivered` | 배달원이 배달 중인 상태 (배달 주문만 해당) | completed |
| 완료 | `completed` | 주문이 정상적으로 완료된 상태 | - |
| 취소 | `cancelled` | 주문이 취소된 상태 | - |


상태 변경은 순방향으로만 진행된다. 예를 들어 `preparing` 상태에서 `pending`으로 되돌릴 수 없다. 주문 상세 페이지에서 "다음 상태로 변경" 버튼을 클릭하면 현재 상태의 바로 다음 단계로 변경된다.

## 금액 구성

주문 금액은 다음과 같이 구성된다.

```
최종 결제금액 = 상품 합계 - 쿠폰 할인 - 포인트 사용 + 배달비
```

- **상품 합계** (`subtotalAmount`): 주문한 메뉴의 단가 × 수량의 합. 옵션 가격도 포함된다.
- **쿠폰 할인** (`discount.couponAmount`): 쿠폰을 적용한 경우의 할인 금액. 사용한 쿠폰 이름도 함께 저장된다.
- **포인트 사용** (`discount.pointUsed`): 적립 포인트를 사용한 금액.
- **배달비** (`deliveryFee`): 배달 주문에만 부과되며, 포장/매장식사는 0원이다.
- **최종 결제금액** (`totalAmount`): 고객이 실제로 결제한 금액.

## 현금영수증

현금 결제 시 고객이 현금영수증을 요청할 수 있다.

| 필드 | 설명 |
| --- | --- |
| `requested` | 발급 요청 여부 (`true`/`false`) |
| `type` | 소득공제(`income_deduction`) 또는 지출증빙(`expense_proof`) |
| `number` | 발급 번호 (전화번호 또는 사업자등록번호) |

## 주문 취소

관리자는 `pending`(결제 대기) 또는 `confirmed`(주문 확인) 상태의 주문만 취소할 수 있다. 준비 중 이후 상태의 주문은 취소 버튼이 표시되지 않는다.

취소 시 사유를 선택해야 한다.

| 취소 사유 | 코드 | 설명 |
| --- | --- | --- |
| 고객 요청 | `customer_request` | 고객이 직접 취소를 요청한 경우 |
| 재료 소진 | `out_of_stock` | 메뉴 재료가 소진되어 조리할 수 없는 경우 |
| 매장 마감 | `store_closed` | 매장이 영업 종료 또는 임시 휴업인 경우 |
| 기타 | `other` | 위에 해당하지 않는 사유 |


취소 사유 외에 상세 사유를 텍스트로 추가 입력할 수 있다 (선택 사항). 취소 처리 시 처리자 이름과 취소 일시가 자동으로 기록된다.

---

## 주문 메모

관리자는 주문에 메모를 추가할 수 있다. 메모는 주문 상세 페이지의 오른쪽 패널에 시간순으로 표시된다. 각 메모에는 내용, 작성자, 작성 시간이 기록된다.

메모 입력란에서 Enter 키를 누르거나 전송 버튼을 클릭하면 메모가 추가된다. 메모 수정이나 삭제 기능은 제공하지 않는다.

---

## 주문 목록 페이지

### 통계 카드

페이지 상단에 4개의 통계 카드가 표시된다.

| 카드 | 설명 |
| --- | --- |
| 전체 주문 | Mock 데이터 기준 전체 주문 건수 |
| 대기 중 | `pending` 상태인 주문 건수 |
| 오늘 완료 | 오늘 날짜에 `completed`된 주문 건수 |
| 오늘 취소 | 오늘 날짜에 `cancelled`된 주문 건수 |


### 검색 필터

6개의 필터 조건을 조합하여 주문을 검색할 수 있다.

| 필터 | 입력 방식 | 설명 |
| --- | --- | --- |
| 시작일 | 날짜 선택 | 주문 일시 시작 범위 |
| 종료일 | 날짜 선택 | 주문 일시 종료 범위 (해당일 23:59:59까지 포함) |
| 주문유형 | 드롭다운 | 전체, 배달, 포장, 매장식사 |
| 상태 | 드롭다운 | 전체, 결제 대기, 주문 확인, 준비 중, 준비 완료, 배달 중, 완료, 취소 |
| 가맹점 | 드롭다운 | 전체, 강남점, 홍대점, 잠실점, 신촌점, 건대점 |
| 검색 | 텍스트 | 주문번호, 주문자명, 전화번호로 키워드 검색 |


필터를 변경하면 페이지가 1페이지로 초기화된다. "초기화" 버튼을 클릭하면 모든 필터가 기본값으로 돌아간다.

### 주문 테이블

필터 조건에 맞는 주문이 테이블로 표시된다. 컬럼 구성은 다음과 같다.

| 컬럼 | 설명 |
| --- | --- |
| 주문번호 | `ORD-YYYYMMDD-NNN` 형식. 모노스페이스 폰트로 표시 |
| 일시 | 주문 접수 일시 |
| 유형 | 배달/포장/매장식사 뱃지 |
| 주문자 | 주문자 이름 |
| 가맹점 | 가맹점 이름 |
| 메뉴 | 첫 번째 메뉴명(수량) + "외 N건" 형태로 요약 |
| 결제금액 | 최종 결제금액 (원 단위) |
| 결제수단 | 카드, 카카오페이 등 |
| 상태 | 상태별 색상 뱃지 (pending=노랑, completed=초록, cancelled=빨강 등) |
| 상세 | 상세보기 아이콘 버튼 |


행을 클릭하면 해당 주문의 상세 페이지(`/orders/:id`)로 이동한다. 페이지네이션은 20건 단위이다.

테이블 헤더에는 오늘 매출 합계도 함께 표시된다.

### 엑셀 다운로드

검색 필터 영역의 "엑셀 다운로드" 버튼을 클릭하면, 현재 필터 조건에 맞는 전체 주문 데이터를 엑셀 파일로 다운로드한다. 페이지네이션과 무관하게 조건에 맞는 모든 데이터가 포함된다.

엑셀 파일에 포함되는 컬럼: 주문번호, 주문일시, 주문유형, 주문자, 전화번호, 가맹점, 메뉴, 결제수단, 상품합계, 할인금액, 쿠폰, 포인트사용, 배달비, 결제금액, 상태.

파일명은 `주문목록_YYYY-MM-DD.xlsx` 형식이다.

## 주문 상세 페이지

### 헤더

주문번호, 주문 일시, 상태 뱃지가 표시된다. 오른쪽에는 상태 변경 버튼과 취소 버튼이 있다.

- **상태 변경 버튼**: 현재 상태의 다음 단계로 변경한다. `completed`와 `cancelled` 상태에서는 표시되지 않는다.
- **주문 취소 버튼**: `pending` 또는 `confirmed` 상태에서만 표시된다. 클릭하면 취소 사유 입력 모달이 열린다.

### 왼쪽 영역 (2/3 너비)

#### 주문 기본 정보

주문유형, 주문채널, 주문자(이름+전화번호), 가맹점이 그리드 형태로 표시된다. 고객 요청사항이 있으면 노란색 박스로 별도 표시된다.

#### 주문 메뉴 테이블

주문한 메뉴를 카테고리, 메뉴명, 옵션, 수량, 단가, 소계 컬럼의 테이블로 표시한다. 옵션이 있으면 `옵션명(+가격)` 형태로, 없으면 `-`로 표시된다.

#### 결제 정보

금액 구성을 상세하게 보여준다.

- 상품 합계
- 쿠폰 할인 (사용한 쿠폰 이름과 함께, 빨간색 마이너스)
- 포인트 사용 (빨간색 마이너스)
- 배달비 (배달 주문만)
- 최종 결제금액 (굵게, primary 색상)
- 결제수단
- 현금영수증 (요청한 경우만 표시: 소득공제/지출증빙 + 발급번호)

#### 취소 정보

주문이 취소된 경우에만 표시된다. 취소 사유, 상세 사유, 처리자, 취소 일시를 보여준다.

### 오른쪽 영역 (1/3 너비)

#### 주문 메모

메모 목록이 시간순으로 표시된다. 각 메모에는 내용, 작성 시간, 작성자가 표시된다. 하단에 메모 입력란과 전송 버튼이 있다. 메모가 없으면 "등록된 메모가 없습니다" 안내가 표시된다.

### 취소 모달

주문 취소 버튼을 클릭하면 모달이 열린다. 모달에는 주문번호가 표시되고, 취소 사유 드롭다운(4개 옵션)과 상세 사유 입력란(선택)이 있다. "주문 취소" 버튼을 클릭하면 취소가 처리되고, 성공 시 토스트 메시지가 표시된다.

## 데이터 흐름

```
OrderList.tsx / OrderDetail.tsx (페이지)
  → useOrderList / useOrder / useOrderStats / ... (React Query 훅)
    → orderService (서비스)
      → mockOrderData (Mock 데이터 20건)
```

페이지 컴포넌트는 `useOrders.ts`에 정의된 React Query 훅만 사용하며, 서비스 레이어를 직접 호출하지 않는다.

### 훅 목록

| 훅 | 쿼리 키 | 용도 |
| --- | --- | --- |
| `useOrderList(params)` | `['orders', 'list', params]` | 주문 목록 조회 (필터 + 페이지네이션) |
| `useOrder(id)` | `['order', id]` | 주문 단건 상세 조회 |
| `useOrderStats()` | `['orders', 'stats']` | 통계 카드 데이터 |
| `useCancelOrder()` | mutation | 주문 취소. 성공 시 `['orders']`, `['order', id]` 캐시 무효화 |
| `useUpdateOrderStatus()` | mutation | 상태 변경. 성공 시 `['orders']`, `['order', id]` 캐시 무효화 |
| `useAddOrderMemo()` | mutation | 메모 추가. 성공 시 `['order', id]` 캐시 무효화 |
| `useOrdersForExport(params)` | `['orders', 'export', params]` | 엑셀 다운로드용 전체 조회. `enabled: false`로 수동 트리거 |

## Mock 데이터

20건의 주문 데이터가 `src/lib/api/mockOrderData.ts`에 정의되어 있다. 다양한 주문유형(배달/포장/매장식사), 채널(앱/키오스크/POS/웹), 결제수단, 상태, 가맹점(5개)을 포함한다.

주문 날짜 범위는 2026-02-07 ~ 2026-02-12이며, 메뉴 카테고리는 치킨, 사이드, 음료, 세트메뉴를 포함한다. 취소된 주문 3건(고객 요청, 재료 소진, 매장 마감)과 메모가 포함된 주문도 있다.

## API 엔드포인트

현재는 Mock 서비스로 동작하며, 향후 백엔드 연동 시 다음 엔드포인트를 사용한다.

| Method | Endpoint | 설명 |
| --- | --- | --- |
| `GET` | `/api/orders` | 주문 목록 조회 (필터, 페이지네이션) |
| `GET` | `/api/orders/:id` | 주문 상세 조회 |
| `GET` | `/api/orders/stats` | 주문 통계 조회 |
| `GET` | `/api/orders/export` | 엑셀용 전체 조회 (페이지네이션 없음) |
| `PATCH` | `/api/orders/:id/cancel` | 주문 취소 (사유 포함) |
| `PATCH` | `/api/orders/:id/status` | 주문 상태 변경 |
| `POST` | `/api/orders/:id/memos` | 주문 메모 추가 |


### 목록 조회 파라미터

| 파라미터 | 타입 | 설명 |
| --- | --- | --- |
| `dateFrom` | `Date` | 주문일 시작 범위 |
