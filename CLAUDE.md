# AI 규칙
- **항상 출력 토큰 통제**: 대화 응답 시 "3줄 요약" 형태 또는 사용자 의도에 따라 "결과 필수값만 포함한 짧은 답변/JSON" 형식으로 반환. TMI, 코드 리뷰 통계, 부차적인 배경 설명 금지.
- **전체 스캔 금지**: 무관한 파일/폴더 자동 전체 스캔 금지. 핀포인트 탐색만 수행.
- **문서 읽기 최소화**: `docs/` 폴더 내 마크다운 파일 자동 읽기 금지(요청 시에만).
- **에이전트 단일화**: 서브에이전트 생성 금지. 메인 프로세스에서 단독 처리.

# 핵심 규칙
- 코드만 출력, 불필요한 설명 금지
- 현재 작업 카테고리 폴더 파일만 읽기
- 작업 완료 시 반드시 `docs/progress.md` 업데이트
- 모르면 추측하지 말고 질문할 것

# 작업 방식
- 한 번에 하나의 카테고리만 처리
- 파일 읽기 전 반드시 필요한지 확인
- 전체 파일 읽기 금지 → 필요한 함수/섹션만 읽기
- 백엔드 API 연동을 고려한 코드 작성

# 출력 규칙
- 설명 텍스트 최소화
- 변경된 코드 부분만 출력
- 완료 시 "✅ cat_XX 완료" 한 줄만 출력

# 경량 QA
1. **터미널 결과 의존**: 파일 업데이트 후 `npm run build` 에러 로그만 보고 직접 수정(원본 파일 통째 분석 지양). 결과가 정상이면 최소 텍스트로 보고.
2. **컨벤션 자가점검**:
   - 컴포넌트 PascalCase, 함수 camelCase, 상수 UPPER_SNAKE_CASE, 폴더 kebab-case
   - API fetch는 Service/Hook에서만 처리 (Clean Architecture)
   - `any` 금지, `@/` 절대 경로 사용
3. **핵심 검증**: Entity ↔ Form 매핑, 필수 유효성 체크, Toast 처리 등 필수 로직만 단건 확인.
4. **QA 결과 출력**: 버그 발견 시 "❌ 버그: [내용]", 패스 시 "✅ QA 통과" 한 줄만 출력.
