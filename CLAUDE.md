# AI 규칙
- **출력 토큰 통제**: 3줄 요약 / 결과 필수값만. TMI, 부차적 배경 설명 금지.
- **전체 스캔 금지**: 무관한 파일/폴더 자동 전체 스캔 금지. 핀포인트 탐색만 수행.
- **문서 읽기 최소화**: `docs/` 폴더 내 마크다운 파일 자동 읽기 금지(요청 시에만).
- **정의된 에이전트만 사용**: `.claude/agents/`에 정의된 에이전트만 호출. 임의 서브에이전트 생성 금지.

# 작업 원칙 (PDCA)
1. **기획/설계 확인** — 기능 구현 전 반드시 기획서 확인 후 계획 수립
2. **코드 작성** — 계획에 따라 구현, 기획에 없는 해석 금지
3. **소스 리뷰** — 구현 완료 후 코드 품질/아키텍처 점검
4. **수정** — 리뷰 결과에 따라 수정 반영

# 핵심 룰
- 기획에 없는 해석 금지 (임의 기능/동작 추가 금지)
- 백엔드 API 연동과 확장성을 고려하여 작성
- 코드만 출력, 불필요한 설명 금지
- 현재 작업 카테고리 폴더 파일만 읽기
- 작업 완료 시 `docs/progress.md` 업데이트
- 모르면 추측하지 말고 질문할 것
- 작업 전 수정 파일 경로를 먼저 파악하고 기록

# 출력 규칙
- 설명 텍스트 최소화, 변경된 코드 부분만 출력
- 완료 시 "완료" 한 줄만 출력

# currentDate
