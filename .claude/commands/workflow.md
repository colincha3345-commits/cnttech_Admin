# 멀티 에이전트 워크플로우

사용자 요청: $ARGUMENTS

## 실행 절차

아래 단계를 순차적으로 실행한다. 각 단계의 산출물은 다음 단계의 입력으로 전달한다.

### 1단계: 제품 기획
- **에이전트**: `product-planner`
- **입력**: 사용자 요청 (`$ARGUMENTS`)
- **작업**: 요청을 분석하여 PRD 초안 작성
- **산출물**: 기획서 초안

### 2단계: UX 보완 + 아이디어 (병렬)
두 에이전트를 **동시에** 실행:

**에이전트 A**: `ux-planner`
- **입력**: 1단계 기획서 초안
- **작업**: 사용자 플로우, 화면 구성, UI 상태, 인터랙션 설계

**에이전트 B**: `idea-generator`
- **입력**: 1단계 기획서 초안
- **작업**: 엣지케이스 발굴, 개선 아이디어, 확장성 검토

### 3단계: 기획 통합
- 1~2단계 결과를 종합하여 **최종 기획서** 확정
- UX 보완 사항과 아이디어 중 P0/P1을 기획서에 반영
- 사용자에게 최종 기획서를 보여주고 **승인 요청**

### 4단계: 디자인 리뷰 (선택)
- **에이전트**: `design-reviewer`
- **조건**: .pen 파일이 있거나 UI 변경이 포함된 경우에만 실행
- **작업**: 디자인 일관성, 사용성, 접근성 점검

### 5단계: 개발 실행
- 확정된 기획서를 기반으로 코드 구현
- Clean Architecture 준수: Page → Hook → Service → API
- 구현 완료 후 `npm run build`로 빌드 검증

### 6단계: 코드 리뷰 + 보안 감사 (병렬)
두 에이전트를 **동시에** 실행:

**에이전트 A**: `code-reviewer`
- **작업**: 아키텍처 준수, 타입 안전성, 코드 품질 점검

**에이전트 B**: `security-auditor`
- **작업**: 권한/인증/데이터 보호 감사

### 7단계: 최종 수정
- 6단계 리뷰 결과에서 CRITICAL/WARNING 이슈 수정
- 수정 후 재빌드 검증
- `docs/progress.md` 업데이트

## 중단 조건
- 3단계에서 사용자가 기획을 거부하면 1단계로 돌아감
- 6단계에서 CRITICAL 이슈가 있으면 5단계로 돌아감
